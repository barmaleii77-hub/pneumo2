# RELEASE NOTES R31AK (2026-03-26)

## Что исправлено

### 1. animator-facing densification для high-speed playback
- `anim_latest` теперь при экспорте может автоматически **уплотняться для Desktop Animator**;
- политика уплотнения двухконтурная:
  - максимум по времени: `1/120 s` на кадр;
  - максимум по пройденному пути: `0.10 m` на кадр;
- discrete/open-state таблицы (`df_open`) при этом ресемплируются **step-hold**, без выдуманных промежуточных состояний клапанов.

### 2. smoothness-first playback policy в Desktop Animator
- при `speed <= x1.0` playback теперь **предпочитает гладкость** перед агрессивным catch-up;
- за один таймерный тик показывается ограниченное число новых sample frames, а оставшийся time debt доедается следующими тиками;
- это уменьшает визуальные прыжки, когда GUI thread кратковременно не успевает.

## Почему это понадобилось
Свежий Windows bundle показал, что при более высоких скоростях `anim_latest` приходил в Animator со слишком крупным шагом по пройденному расстоянию. Это создавало ощущение, что "кадров недостаточно", даже до учёта отдельной проблемы с общей производительностью UI.

## Что сознательно НЕ объявляется закрытым
- общий Desktop Animator GL/layout perf track;
- Web UI idle CPU после детального прогона;
- визуальная упаковка цилиндров/поршней.

R31AK — это точечный corrective release именно по frame density / playback smoothness.
