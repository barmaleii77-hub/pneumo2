# SpringTable: PCHIP и режимы x0 для нулевой позы (v6.50)

Этот документ описывает улучшения **механической части** модели WorldRoad:

1) Табличная пружина с режимом интерполяции **PCHIP** (гладкая, монотонная).
2) Режимы расчёта `x0` — базового сжатия пружины в нулевой позе.
3) Дополнительные параметры для полного «smooth»-режима и диагностики.

> Важно: если вы ничего не включаете, поведение остаётся как раньше (legacy по умолчанию).

---

## 1) Интерполяция табличной пружины

Файл: `pneumo_solver_ui/spring_table.py`.

Параметр модели:

- `spring_interp_mode`: `'linear'` (по умолчанию) или `'pchip'`.

Что меняется:

- `'linear'`: используется `np.interp`.
- `'pchip'`: используется монотонная кубическая интерполяция (PCHIP), даёт гладкую кривую без паразитных осцилляций между узлами таблицы.

Рекомендация:

- Для обычных расчётов и воспроизводимости старых результатов — `'linear'`.
- Для градиентной оптимизации и «полностью гладкой» модели — `'pchip'`.

---

## 2) Режимы `spring_x0_mode` (расчёт статического положения)

`x0` — базовое сжатие пружины в нулевой позе при ровной дороге.

Параметр:

- `spring_x0_mode`: `'global'|'axle'|'corner'|'manual'`

### 2.1) `global` (по умолчанию)

Один `x0` на все 4 колеса. Близко к старому алгоритму (среднее по углам).

### 2.2) `axle`

Два значения: `x0_front` и `x0_rear`.

Полезно, если:

- motion ratio перед/зад отличается,
- вы хотите, чтобы нулевая поза была ближе к физической реальности без «средней» геометрии.

### 2.3) `corner`

Четыре значения: по каждому углу (`ЛП/ПП/ЛЗ/ПЗ`).

Это наиболее «точный» режим по статике, если:

- есть асимметрия по геометрии,
- есть асимметрия по давлению/массе/параметрам.

### 2.4) `manual`

Ручная установка `x0`:

- `spring_x0_m` (alias: `пружина_x0_м`) может быть:
  - число (одно значение на все углы)
  - список/массив длины 4 (по углам в порядке `ЛП,ПП,ЛЗ,ПЗ`).

## 3) Связь `x0` с нулевой позой

Цель: при **δ=0** суммарная вертикальная сила должна компенсировать массу.

Упрощённо:

- На каждом углу «целевая» вертикальная сила пружины:
  - `F_spr_vert_target = max(0, m_body*g/4 - F_cyl_vert0)`

Далее переводим в осевую силу пружины:

- Если `механика_пружина_режим == 'delta'`: осевая = вертикальная (legacy)
- Иначе (coilover по Ц1):
  - `F_spr_axial_target = F_spr_vert_target / motion_ratio`,
  - где `motion_ratio = max(eps, -ds1_ddelta0)`

И по таблице находим `x0` (через инверсию `F(x)`):

- `F_table_target = F_spr_axial_target / масштаб_пружины`

## 4) Жёсткий преднатяг на отбое (`пружина_преднатяг_на_отбое_строго`)

Если включено:

- для каждого угла проверяем, что на полном отбое пружина не «разгружается» ниже `пружина_преднатяг_на_отбое_min_м`.
- если нужно — увеличиваем `x0` (по углам) на величину недобора.

Поля диагностики в `df_atm`:

- `spring_preload_strict_applied`
- `spring_preload_slack_rebound_m` (максимальный недобор)
- `spring_preload_slack_rebound_{ЛП/ПП/ЛЗ/ПЗ}_m`

---

## 5) fully_smooth_mode и smooth_stops

Для градиентной оптимизации удобен один флаг:

- `fully_smooth_mode: True`

Он включает сглаживания по умолчанию:

- механика/контакты/кинематика штоков,
- (и если не переопределено) `pneumo_flow_smooth_mode`.

Также добавлен параметр:

- `smooth_stops: True|False` (по умолчанию следует `smooth_mechanics`)

Сглаживает bump/rebound stops (упоры хода), что убирает разрывы производной при входе в упор.

---

## 6) p0_override: переопределение начальных давлений

Иногда нужно жёстко задать начальные давления в отдельных узлах сети.

Параметр:

- `p0_override` можно передать в `params` или в `test`.

Формат:

- dict `{pattern: pressure_Pa}`
- pattern поддерживает `fnmatch`-маски (`*`, `?`).

Пример:

- `{'Ресивер2': 350000, 'Ц1_*_БП': 101325}`

Ограничения:

- Узел `АТМ` не переопределяется (чтобы не расходиться с константой `P_ATM`).

---

## 7) Новые сигналы для анимации/проверки

В `df_main` добавлены:

- `колесо_контакт_{ЛП/ПП/ЛЗ/ПЗ}_м` — высота точки контакта
- `сжатие_шины_{ЛП/ПП/ЛЗ/ПЗ}_м` — `z_road - z_contact`
- `скорость_сжатия_шины_{ЛП/ПП/ЛЗ/ПЗ}_м_с`

И расширена самопроверка (`mech_selfcheck_*`):

- геометрия рамы (roll/pitch)
- связь контакт/центр колеса
- связь «сжатие шины = дорога - контакт»
