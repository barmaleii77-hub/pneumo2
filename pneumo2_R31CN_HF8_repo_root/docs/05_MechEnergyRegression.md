# 05 — Mech Energy Regression (suite)

## Что это

`mech_energy_regression_report.py` — это инструмент, который прогоняет **набор тестов** (suite)
и строит сводную таблицу ошибок по:

1) **Баланс механической энергии** (кузов/колёса/пружины/шина/отбойники/гравитация).
2) Независимая проверка **p·dV** цилиндров: `p_gauge * dV/dt` против `F_cyl * ṡ`.

Цель: ловить регрессии матмодели, когда отдельный тест ещё «выглядит нормально»,
но на другом возбуждении всплывают ошибки знаков/мэппинга/жёсткости.

## Где находится

- Скрипт: `pneumo_solver_ui/tools/mech_energy_regression_report.py`
- Отчёты: `pneumo_solver_ui/reports/`

Всегда обновляются файлы:
- `mech_energy_regression_LAST.md`
- `mech_energy_regression_LAST.json`

(плюс версия с timestamp на каждый запуск).

## Как запускать

Из директории `pneumo_solver_ui`:

```bash
python tools/mech_energy_regression_report.py
```

Строгий режим (rc=2 при провале порогов):

```bash
python tools/mech_energy_regression_report.py --strict
```

Ограничить длительность каждого теста:

```bash
python tools/mech_energy_regression_report.py --t-end-cap 1.2
```

## Что смотреть в отчёте

Колонки:

- `max_rel_err_E` — max |ΔE|/|E| по механо‑балансу (основной индикатор).
- `end_rel_err_E` — относительная ошибка на конце интервала.
- `max_abs_err_E_J` — max |ΔE| в Дж (полезно, если E≈0 и относительная нестабильна).
- `max_pdv_err_W` — max |(p·dV)_gauge − ΣF*ṡ| в ваттах.
- `end_pdv_work_err_J` — интеграл ошибки p·dV (должен быть около 0).

Флаги:

- `FAIL` — превышение порога `rel_th` или `pdv_th`.

## Ограничения и интерпретация

- Это **диагностический** инструмент, а не строгая физическая сертификация.
- В модели есть сглаживания (`smooth_*`), односторонние элементы (контакт/ограничители),
  численные производные (`gradient`) — поэтому «нулевые» ошибки на уровне машинного нуля не всегда достижимы.
- Тем не менее p·dV‑проверка должна быть очень близка к нулю: это чистая проверка согласованности уравнений
  (давление/площадь/знак скорости штока).
