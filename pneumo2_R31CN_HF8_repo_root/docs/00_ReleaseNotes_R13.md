# Obobshchenie R13 — Release Notes

Дата сборки: 2026-01-21

Этот релиз сосредоточен на **устойчивости диагностики** (чтобы вы могли быстро присылать мне полный набор артефактов) и на **удобном выборе логов** для калибровки (NPZ/CSV) без ручного ввода путей и без работы в консоли.

---

## 1) Full diagnostics: больше не падает и всегда собирает ZIP

### Симптомы (по вашим логам)

- `FATAL: run_full_diagnostics crashed`
- `NameError: name 'datetime' is not defined`

### Что сделано

- Добавлен недостающий импорт `from datetime import datetime`.
- Добавлен недостающий хелпер `_copy_tree()` (ранее был вызов без реализации).
- В `argparse` добавлены параметры `--opt_minutes`, `--opt_jobs`, которые использовались, но не объявлялись.
- `main()` теперь работает по схеме **try/except/finally**: даже если часть шагов падает, скрипт:
  - пишет `_FATAL.txt` (если было исключение),
  - **всё равно пытается собрать ZIP** с тем, что успело собраться.

### Результат

`RUN_FULL_DIAGNOSTICS_WINDOWS.bat` снова пригоден как «единый способ» получить полный пакет диагностики одним кликом.

---

## 2) Calibration NPZ: нормальный выбор файлов (NPZ/ZIP), меньше ручных путей

### Симптом/пожелание

> «сделай нормальный выбор файлов в Калибровка по логам (NPZ/CSV)»

### Что сделано

На странице `Calibration NPZ` добавлен блок **"0) Выбор логов (NPZ/CSV) — без ручного ввода путей"**:

- **Загрузка NPZ файлов** (`*.npz`) в диалоге выбора файлов → сохранение в `osc_uploads/UPLOAD_...` → автоматическая подстановка `osc_dir`.
- **Загрузка ZIP** (с NPZ и/или CSV внутри) → распаковка в `osc_uploads/UPLOAD_...` → автоматическая подстановка `osc_dir`.
- Показ статуса набора: сколько NPZ/CSV, есть ли `signals.csv`.
- Если `T*_osc.npz` не найдены — autopilot не запускается «вслепую», а выдаёт понятную ошибку и подсказку про шаг `CSV→NPZ`.

## Известные ограничения

- Streamlit по-прежнему **не умеет выбирать папки** через системный диалог, поэтому для «не вводить пути руками» используется загрузка файлов/ZIP.
- Закрытие вкладки браузера **не останавливает** сервер Streamlit — это нормальная модель работы. Остановка делается `Ctrl+C` в консоли или кнопкой Stop (если включена в Diagnostics).
