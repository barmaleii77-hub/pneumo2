# MechanikaR49: World-road z(x,y) и расширенная наблюдаемость

## 1) Что сделано и зачем

В версии **PnevmatikaR47** профиль дороги задавался как функция времени:

- `road_func(t) -> [z_LP, z_PP, z_LZ, z_PZ]`

Это удобно для «стендовых» сценариев, но имеет два ограничения:

1. **Нет мировой геометрии движения.** Колёса не «едут» по поверхности, а получают готовые значения z(t). Сложно описывать дорожные события в координатах трассы (кочка в точке x=5 м, волна с длиной λ, и т.п.).
2. **В контакте шины игнорировалась скорость дороги.** Использовалось приближение `pen_dot = -z_w_dot`, что физически неверно, если дорога меняется со временем (или эквивалентно: если машина едет по профилю). Это влияет на демпфирование контакта.

В **MechanikaR49** добавлено:

- **World-road режим**: дорога задаётся как **поверхность** `z = z(x, y)` с градиентами `dz/dx`, `dz/dy`.
- **Предвычисление траектории** (кинематика без бокового сноса): `x(t), y(t), yaw(t), v(t)` на основе `ax_func(t)` и `ay_func(t)`.
- **Автогенерация `road_func(t)` и `road_dfunc(t)`** из `z(x,y)` + траектории.
- **Корректное `pen_dot`**: `pen_dot = z_road_dot - z_wheel_dot`.
- **Расширенная наблюдаемость**: добавлены временные ряды положений/скоростей/ускорений рамы в углах, относительных ходов подвески `delta`, проникновения шины и скорости дороги.

Важно: это **не меняет схему пневматики** и не добавляет исполнительных механизмов. Изменения касаются механики, профиля дороги и диагностических сигналов.

## 2) Как включить world-road

World-road активируется автоматически, если в тесте (словарь `test`) присутствует ключ:

- `test['road_surface']` — описание поверхности дороги.

Форматы `road_surface`:

### 2.1. Плоская дорога
```python
road_surface = {'type': 'flat'}
```

### 2.2. Синус по оси X (волнистость)
```python
road_surface = {
  'type': 'sin_x',
  'A': 0.02,          # амплитуда, м
  'lambda_m': 5.0,    # длина волны, м
  'phase': 0.0        # фаза, рад
}
```

### 2.3. C2-гладкая «кочка» вдоль X
```python
road_surface = {
  'type': 'bump_x',
  'A': 0.03,          # высота кочки, м (для ямы задайте отрицательное)
  'x0': 5.0,          # начало кочки, м
  'dur_m': 0.7,       # длина по x, м
  'ramp_ratio': 0.25  # доля длины на подъём (и столько же на спуск)
}
```

### 2.4. Сумма нескольких элементов
```python
road_surface = [
  {'type': 'sin_x', 'A': 0.01, 'lambda_m': 6.0},
  {'type': 'bump_x', 'A': 0.03, 'x0': 5.0, 'dur_m': 0.6},
]
```

### 2.5. Пользовательская функция
```python
def surface(x, y):
    z = 0.02*np.sin(2*np.pi*x/4.0)
    dzdx = 0.02*(2*np.pi/4.0)*np.cos(2*np.pi*x/4.0)
    dzdy = 0.0*x
    return z, dzdx, dzdy

road_surface = surface
```

## 3) Траектория (x(t), y(t), yaw(t), v(t))

Траектория считается кинематически (без бокового сноса) по уравнениям:

- `v_dot = ax(t)`
- `yaw_dot = ay(t) / max(|v|, eps)`  (из связи `ay ≈ v*yaw_dot`)
- `x_dot = v*cos(yaw)`
- `y_dot = v*sin(yaw)`

Это **первый приближённый** слой, цель которого — связать профиль дороги `z(x,y)` с тестовыми `ax_func/ay_func` и получить корректный `z_road_dot`.

Начальные условия траектории можно задать через:

```python
test['world'] = {
  'v0_m_s': 5.0,
  'yaw0_rad': 0.0,
  'x0_m': 0.0,
  'y0_m': 0.0,
}
```

Если `test['world']` не задан, используются нули.

## 4) Контакт шины с дорогой: что изменилось

### 4.1. Проникновение

Добавлена опция интерпретации координаты колеса (`params['колесо_координата']`):

- `'contact'` (по умолчанию): `z_w` — координата пятна контакта.
  - `pen = z_road - z_w`
- `'center'`: `z_w` — координата центра колеса.
  - `pen = z_road + R_w - z_w`, где `R_w = params['радиус_колеса_м']`

### 4.2. Скорость проникновения

Теперь:

- `pen_dot = z_road_dot - z_w_dot`

Ранее было приближение `pen_dot = -z_w_dot`.

Демпфирование контакта остаётся односторонним (только при сжатии):

- если `pen > 0`, то `F = k*pen + c*max(pen_dot, 0)`
- иначе `F = 0`

## 5) Какие новые сигналы появились в результатах

### 5.1. Мир (если активирован road_surface)

- `мир_x_м`, `мир_y_м`
- `мир_yaw_рад`
- `мир_скорость_v_м_с`
- `мир_ax_м_с2`, `мир_ay_м_с2`

### 5.2. Ускорения рамы

- `ускорение_рамы_z_м_с2`
- `угловое_ускорение_phi_рад_с2`
- `угловое_ускорение_theta_рад_с2`

### 5.3. Рама в углах (точки крепления подвески)

Для каждого угла (ЛП/ПП/ЛЗ/ПЗ):

- `рама_z_*_м`
- `рама_zdot_*_м_с`
- `рама_zddot_*_м_с2`

### 5.4. Относительный ход подвески

`delta = z_колеса - z_рамы_в_углу` (как и в основной модели):

- `delta_*_м`
- `delta_dot_*_м_с`
- `delta_ddot_*_м_с2`

### 5.5. Контакт шины

- `проникновение_шины_*_м` (может быть отрицательным, если есть зазор)
- `скорость_проникновения_шины_*_м_с`
- `скорость_дороги_*_м_с`

## 6) Ограничения текущей реализации

1. Траектория world-road — **кинематическая**, без моделирования бокового сноса/шины (slip angle), без динамики yaw вокруг вертикальной оси (момент инерции Izz).
2. `yaw_dot = ay/v` корректно в рамках «идеальной» кинематики при малом сносе. Для физики на пределе сцепления потребуется модель шин (Pacejka/brush) и баланс сил.
3. Параметр `колесо_координата` по умолчанию оставлен `'contact'`, чтобы не ломать совместимость со старой интерпретацией `z_w`. Для учёта диаметра колеса включайте `'center'` и контролируйте согласованность с остальной геометрией.
