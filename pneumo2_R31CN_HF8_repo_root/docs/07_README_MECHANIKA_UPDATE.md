# Mechanika — update над UnifiedPneumoApp v6.25 (rel0 + selfcheck кинематики)

Этот пакет — *механическое* улучшение базовой версии **UnifiedPneumoApp_UNIFIED_v6_25_WINSAFE**.

Ключевая цель обновления: сделать математическую модель и визуализацию **однозначно интерпретируемыми** за счёт:

1) **Нулевой позы (rel0)**: дополнительные колонки `*_rel0` считаются относительно статики на ровной дороге.
   Это удобно для **графиков/анализа**.
   **Закон:** анимация геометрии (Desktop Animator) рисует только **абсолютные** координаты модели.
2) **Автономного sanity-check**: модель сама проверяет корректность всех взаимных перемещений (колесо↔дорога↔рама, шток↔кинематика) и ловит типовые ошибки «перепутали углы/колёса/штоки».

Подробно: `docs/04_MechRel0Selfcheck.md`.

---

## Что именно сделано

### 1) rel0-колонки (нулевая поза) в механической модели

Файл:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`

Изменение:
- После построения `df_main` автоматически добавляются колонки `*_rel0` для всех численных сигналов, заканчивающихся на:
  - `_м` (метры)
  - `_рад` (радианы)

То есть, например:
- `перемещение_колеса_ЛП_м_rel0`
- `дорога_ПЗ_м_rel0`
- `крен_phi_рад_rel0`

Опция:
- `use_rel0` (по умолчанию `True`).

### 2) Санити‑чек кинематики (автономная проверка)

Файл:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`

Добавлено:
- вычисляются ошибки самосогласованности:
  - колесо относительно рамы: `(z_wheel - z_frame_corner) - z_rel`;
  - колесо относительно дороги: `(z_wheel - z_road) - z_rel`;
  - шток Ц1/Ц2: сравнение логируемого `s(t)` с пересчётом из `delta(t)` и геометрии double‑wishbone.
- добавлены эвристики, чтобы ловить «все колёса одинаковые» при наличии `ax/ay`.

Результат:
- метрики пишутся в `df_atm` (одна строка) как колонки `mech_selfcheck_*`.

Опции:
- `mechanics_selfcheck` (по умолчанию `True`)
- `mechanics_selfcheck_strict` (по умолчанию `False`) — если включить, будет `raise` при ошибке выше допуска
- `mechanics_selfcheck_tol_m` (по умолчанию `1e-6` м)

### 3) Графики в UI автоматически показывают rel0, если он есть

Файл:
- `pneumo_solver_ui/pneumo_ui_app.py`

Изменение:
- `plot_lines()` теперь, если в данных есть колонка `X_rel0`, а пользователь выбрал `X`, будет **рисовать `X_rel0`**, но в легенде оставит имя `X`.

Ключ в session_state:
- `use_rel0_for_plots` (по умолчанию `True`).

В UI (sidebar) добавлен чекбокс **"Графики: rel0 (обнуление перемещений на t=0)"**.

### 4) Анимация mech_anim: штоки показываются как Δ/abs

Файл:
- `pneumo_solver_ui/components/mech_anim/index.html`

Изменение:
- текст штоков теперь в формате: `Δ / abs` (мм), где `Δ` отсчитана от статики (t=0).

### 5) Desktop Animator (NPZ): абсолютная геометрия как закон

Файл:
- `pneumo_solver_ui/desktop_animator/data_bundle.py`

Закон:
- `DataBundle.get()` **не подставляет** `*_rel0` автоматически.
- Анимация рисует геометрию строго по **абсолютным** сигналам модели.
- `*_rel0` остаются для графиков/анализа и используются только явно.

## Как проверить быстро

1) Установить зависимости:

```bash
python -m pip install -r requirements.txt
```

2) Запустить UI:

```bash
streamlit run pneumo_solver_ui/pneumo_ui_app.py
```

3) Выбрать модель:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`

4) Запустить любой тест и проверить:
- графики перемещений стартуют с нуля;
- в `df_atm` есть `mech_selfcheck_ok = 1`;
- в анимации колёса/рама/дорога отображаются в **абсолютной** геометрии модели (без скрытых baseline);
- штоки отображаются как `Δ/abs`.
