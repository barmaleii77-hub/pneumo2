# RELEASE NOTES — Obobshchenie R34

Дата: 2026-01-23

## Главное
Этот релиз — фикс стабильности UI вокруг **детального прогона (full‑лог)** и диагностики.

## Исправления
1) **Краш при старте/рендере детального блока**
- Исправлено: `NameError: detail_dt is not defined` в месте сборки `cache_key` детального кэша.
- Причина: в R33 в ключ кэша добавили `dt/t_end`, но забыли вычислить их до использования.

2) **Стейт виджетов детального прогона**
- `Макс точек (downsample)` теперь имеет явный `key="detail_max_points"`.
- `Использовать record_full` теперь имеет явный `key="detail_want_full"`.
Это снижает риск того, что Streamlit сопоставит состояние не тому виджету и внезапно поменяет `max_points` (что ранее приводило к рассинхрону кадров/точек и “скачкам” анимации).

3) **NPZ export (все тесты)**
- В helper‑функции генерации NPZ для всех тестов убраны ссылки на `detail_dt/detail_t_end` (которые в той области видимости не гарантированы).
- Исправлена опечатка: `_want_full` → `want_full`.

4) **self_check.py / кодировка Windows**
- В `self_check.py` заменён символ `≈` в `print()` на ASCII (`~`), чтобы скрипт не падал в консолях с `cp1251`.

## Что НЕ решено (важно)
- “Play дёргает ползунок / рассинхрон кадры‑ряды / GPU 50% при открытии вкладки Анимация” — в этом релизе только устранены краши и стабилизирован state. Диагностика/чинёж рерун‑петель и связки playhead↔датасет — следующий шаг (см. `docs/11_TODO.md`).
