# Obobshchenie R39 — Release Notes

Дата: 2026-01-24

## TL;DR

- Исправлена ключевая причина повторных пересчётов: **единый формат ключа кэша детального прогона** во всех сценариях (одиночный, all-tests, экспорт).
- Анимация стала «честнее»: если солвер не экспортирует колонки дороги в лог, профиль дороги берётся **из входного описания теста** (road_func), а не рисуется «как‑то».
- В комплект добавлены **diff/patch** файлы.

## Детали изменений

### 1) Единый (канонический) ключ кэша детального прогона

**Проблема:** в разных местах приложения использовались *разные* форматы ключей:
- одиночный детальный прогон использовал ключ с dt/t_end/mp/full;
- «прогон всех тестов» и некоторые экспорты — упрощённый ключ без dt/t_end.

Это приводило к «промахам» по кэшу и эффекту, когда после «готово» при очередном rerun (например, от playhead) приложение начинало новый расчёт, потому что кэш «не находился».

**Решение:**
- добавлена функция `make_detail_cache_key(...)`;
- все места, где формируется ключ детального прогона, переведены на неё.

Файл: `pneumo_solver_ui/pneumo_ui_app.py`

### 2) Анимация: восстановление профиля дороги из входного теста (road_func)

**Проблема:** не все версии солвера экспортируют дорожный профиль в результирующий лог (`df_main`).
Из‑за этого на тестах `кочка_*`/`микро_*` анимация могла показывать «плоскую дорогу», хотя колёса/подвеска двигались.

**Решение:**
- добавлен helper `compute_road_profile_from_suite(...)`;
- если в `df_main` нет колонок `дорога_*_м`, дорожный профиль вычисляется из suite‑описания теста
  (через `model_mod._compile_suite_test_inputs(...).road_func`).

Важно: **это не “подрисовка физики”**.
- Профиль дороги — это **вход теста**.
- Крен/тангаж/ходы/давления — берутся **только из результатов расчёта**.

Файл: `pneumo_solver_ui/pneumo_ui_app.py`

### 3) Диффы/патчи

Добавлена папка `diffs/`:
- `R39_from_R38.patch` — компактный патч (только изменения R39 поверх R38)
- `R39_from_R35.patch` — полный патч от R35 до R39

## Что дальше (следующий шаг)

1) Довести «анимация ↔ графики» до режима без тяжёлых rerun’ов (фронтенд‑проигрывание по умолчанию, сервер‑sync опционально).
2) Провести аудит матмодели по пунктам:
   - влияние радиуса колеса (проверка параметра `радиус_колеса_м` во всех режимах),
   - распределение момента/тормоза между осями,
   - валидация крена/дифферента на эталонных сценариях.
