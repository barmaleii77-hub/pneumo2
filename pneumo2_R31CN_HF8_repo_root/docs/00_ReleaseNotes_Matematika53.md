# Matematika53 — Release Notes (2026-01-25)

Релиз **Matematika53** развивает ветку Matematika52 и добавляет **механический энерго‑аудит** (в дополнение к уже существующему газо‑/тепло‑/эксерго‑аудиту).

---

## 1) Главное изменение: механический энерго‑аудит

Добавлен энерго‑баланс **механической подсистемы**:

- кузов: `z, φ, θ` и соответствующие скорости,
- 4 колеса: `zw[4]` и `zw_dot[4]`,
- пружины: табличная нелинейность `F(x)` (с масштабом),
- шины: `k_tire` + односторонний демпфер `c_tire`,
- отбойники: `k_stop` + односторонний демпфер `c_stop`.

Баланс строится по внешним работам/потерям:

- **Работа дороги (base excitation)**  
  `W_road = ∫ Σ F_tire · z_road_dot dt`  
  где `F_tire = k_tire*pen + c_tire*max(pen_dot,0)` при `pen>0`.

- **Работа инерционных моментов от ax/ay**  
  `W_axay = ∫ [ (m·ay·h_cg)·φ_dot + (-m·ax·h_cg)·θ_dot ] dt`

- **Работа пневматики по gauge**  
  `W_pneumo_gauge = ∫ Σ (p - P_ATM) · dV dt`  
  Это согласовано с тем, что силы цилиндров в ОДУ вычисляются как `(p - P_ATM)*A`.

- **Потери**  
  `D_tire = ∫ Σ c_tire · max(pen_dot,0)^2 dt`  
  `D_stop = ∫ Σ c_stop · max(v_rel,0)^2 dt` (отдельно для верхнего/нижнего отбойника)

Итого проверяется баланс:

`E_mech(t) ≈ E_mech(0) + W_road + W_axay + W_pneumo_gauge - D_tire - D_stop`

---

## 2) Нюанс p·dV: absolute vs gauge

Газовый баланс (1‑й закон) использует **absolute** работу `p_abs·dV`.  
Механический баланс использует **gauge** работу `(p_abs - P_ATM)·dV`.

Разность логируется:

`p_dV_abs_minus_gauge = ∫ P_ATM · dV_total dt`

Это помогает диагностировать «пограничную» работу атмосферы (типичный эффект single‑rod цилиндров и разности эффективных площадей).

## 5) Рекомендации по интерпретации

- Если остаток мех.баланса «прыгает» при контакте/ударе:
  - уменьшайте `dt` / повышайте `n_substeps`,
  - используйте гладкую дорогу (`road_func_dot`) или задавайте аналитическую производную,
  - контролируйте жёсткость/демпфирование отбойников.

- Если остаток стабильно имеет тренд:
  - проверьте согласованность знаков (дорога/penetration),
  - проверьте, что `road_func_dot` корректен,
  - смотрите `p_dV_abs_minus_gauge_Дж` как источник систематической разницы (absolute vs gauge).
