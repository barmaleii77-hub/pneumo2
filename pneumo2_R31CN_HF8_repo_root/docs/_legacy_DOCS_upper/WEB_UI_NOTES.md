# WEB_UI_NOTES (R02)

## Основная цель
Сделать веб‑интерфейс **устойчивым**:
- пользователь может «жать куда попало»;
- обновлять страницу (F5/Ctrl+F5);
- закрывать браузер;
- перезапускать приложение.

…и при этом **введённые им значения** не должны исчезать.

## Почему понадобились autokeys
В исходниках много Streamlit‑виджетов без явного `key=`.

Streamlit хранит состояние в `st.session_state` и привязывает его к ключам виджетов.
Если ключи неявные, то:
- автосохранение (которое фильтрует ключи по префиксам) не может надёжно «угадать»,
  что именно нужно сохранять;
- при перестановке блоков UI/изменении текста могут меняться внутренние идентификаторы;
- в многостраничных сценариях возрастает риск потери состояния.

Решение в R02: модуль `pneumo_solver_ui/ui_autokeys.py` добавляет стабильные ключи
автоматически (best‑effort) и делает поведение предсказуемым.

## Почему кнопки не сохраняем
Кнопка — это триггер действия. Если сохранить её состояние и восстановить при
старте, можно получить «самонажатие».

Поэтому автоключи кнопок имеют префикс `btn_...`, который исключён из автосохранения.

## Про автосохранение (ui_persistence)
Автосохранение хранит только **легковесные** значения:
- bool/int/float/str (включая `numpy` scalar → конвертация в python тип);
- небольшие dict/list/tuple;
- таблицы‑вводы (DataFrame) только:
  - для известных ключей (`df_params_edit`, `df_suite_edit`, `spring_table_df`),
  - или для `ui_auto_data_editor_...` при малом размере.

Кэш/тяжёлые результаты/фигуры не сохраняются.

## Проверка регрессий
- `python -m pneumo_solver_ui.ui_selfcheck`
- либо страница **Диагностика** → кнопка **ui_selfcheck (UI/UX аудит)**

Отчёт содержит:
- проверку путей навигации;
- наличие ключевых UI модулей;
- проверку, что bootstrap подключает autokeys;
- покрытие важных ключей автосохранением;
- аудит виджетов без явного key= (информативно).
