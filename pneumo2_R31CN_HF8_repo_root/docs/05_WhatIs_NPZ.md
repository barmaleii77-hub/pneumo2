# Что такое NPZ

`.npz` — это **zip‑архив**, который содержит один или несколько файлов формата `.npy` (NumPy arrays).
По сути это удобный контейнер, чтобы хранить **набор массивов** (временные ряды, каналы датчиков и т.д.) одним файлом.

В нашем проекте NPZ используется как «универсальный формат логов» для калибровки/автопилота:

- **симулятор** может выгрузить расчётные логи в `workspace/osc/T01_osc.npz`, `workspace/osc/T02_osc.npz`, ...
- **калибровка / autopilot** затем читают эти файлы как входные данные.

## Что внутри NPZ

Типичный NPZ содержит:

- `t` — время, сек
- `signals` — словарь/набор каналов: `z_FL`, `z_FR`, `p_front`, `p_rear`, `ay`, ...

В NumPy это выглядит так:

```python
import numpy as np

data = np.load("T01_osc.npz")
print(data.files)       # список имён массивов в архиве
t = data["t"]           # numpy.ndarray
z_fl = data["z_FL"]     # numpy.ndarray
```

## Как создаётся NPZ

NPZ можно записать так:

```python
import numpy as np

np.savez(
    "example.npz",
    t=np.array([0.0, 0.1, 0.2]),
    z_FL=np.array([0.0, 0.01, 0.02]),
    z_FR=np.array([0.0, 0.01, 0.02]),
)
```

Для экономии места можно использовать `np.savez_compressed(...)`.

## Заметка про совместимость

NPZ — «тупой» контейнер: он не знает ничего про единицы измерения/метаданные.
Поэтому в проекте есть **mapping (сопоставление каналов)**, чтобы из разных логов (CSV/NPZ/реальные замеры)
делать единый набор сигналов для калибровки.
