# Release Notes — R14

Дата сборки: 2026‑01‑21

## 1) Ключевые изменения

### 1.1 Play в fallback‑анимации 2D/3D стал «человечным»
В прошлой логике `Play` в fallback‑анимации мог восприниматься как «вечный расчёт», потому что Streamlit делает анимацию через **частые rerun‑ы** скрипта (на каждом кадре всё приложение перерисовывается). Это не «зависание», но выглядит как постоянный расчёт.

В R14:
- **Loop по умолчанию выключен** (анимация доходит до конца и останавливается).
- **FPS по умолчанию снижен** (по умолчанию 8 fps), чтобы не превращать UI в постоянный rerun.
- Если вдруг `streamlit-autorefresh` не установлен, автоплей **не делает жёсткий HTML reload** (который может сбрасывать сессию), а выводит понятное сообщение.
- Добавлено **логирование событий play/pause/stop** (через `log_event`, если он доступен в `st.session_state`).

### 1.2 Убран поток предупреждений Streamlit про `use_container_width`
В новых версиях Streamlit параметр `use_container_width` постепенно заменяется на `width="stretch"`.

В R14:
- Добавлены обёртки `safe_dataframe`, `safe_plotly_chart`, `safe_image` — они используют `width="stretch"`, а если версия Streamlit старее — автоматически падают назад на `use_container_width=True`.

### 1.3 «Полный лог» теперь можно считать и экспортировать почти без ручных кликов
Твой запрос: «почему я должен на каждый тест нажимать посчитать полный лог? почему не калибруется автоматом?»

В R14:
- В блоке **«Графики и анимация (baseline)»** добавлено:
  - `Авто‑расчёт полного лога при выборе теста` — если включено, то при выборе теста система сама досчитает детальный лог, если его ещё нет в кэше.
  - `Авто‑экспорт NPZ` — после расчёта детального лога автоматически сохраняется NPZ в `pneumo_solver_ui/workspace/osc` в формате `T01_osc.npz`, `T02_osc.npz`, …
  - Кнопки:
    - **Посчитать полный лог для ВСЕХ тестов**
    - **Экспорт NPZ для ВСЕХ тестов**

Это напрямую закрывает проблему «автопилот ищет T01_osc.npz, а приложение их не генерирует»: теперь UI умеет генерировать NPZ туда, где пайплайны их ожидают.

### 1.4 Добавлен UI‑блок «Калибровка и Autopilot (NPZ/CSV)»
Раньше пайплайны калибровки/автопилота существовали как скрипты, но в UI не было нормального «выбора файлов».

В R14 внизу приложения добавлен expander:
- показывает папку `workspace/osc` и список файлов;
- даёт **загрузить NPZ/CSV** прямо из браузера;
- умеет сделать простую конверсию **CSV → NPZ** (минимальный формат: `main_cols/main_values`);
- запускает:
  - `calibration/pipeline_npz_oneclick_v1.py` (калибровка)
  - `calibration/pipeline_npz_autopilot_v19.py` (autopilot)

### 1.5 Добавлена кнопка «Диагностика (ZIP для отправки)» прямо в UI
Теперь не нужно вручную собирать папку:
- Внизу приложения есть expander **«Диагностика (ZIP для отправки)»**.
- Нажимаешь — получаешь zip, куда попадают:
  - `pneumo_solver_ui/logs`
  - `pneumo_solver_ui/results`
  - `pneumo_solver_ui/calibration_runs`
  - `pneumo_solver_ui/workspace`
  - снимок текущих `base/suite/ranges` как JSON
  - краткий слепок `session_state_snapshot.json`

## 2) Что это даёт относительно твоих вопросов

### «Play — это глобальный playhead или fallback?»
Если у тебя не загружаются веб‑компоненты (Three.js/Canvas), то включается **fallback Matplotlib**. В нём `Play` работает через rerun‑ы Streamlit. Именно поэтому это выглядит как «постоянный расчёт».

В R14 fallback‑play стал:
- без вечного loop по умолчанию,
- с меньшим FPS,
- с логированием start/pause/stop.

### «Почему не логируешь?»
Теперь действия play/pause/stop пишутся через `log_event` (если доступен), иначе молча пропускаются.

### «Почему автопилот/калибровка ищут папки/имена, которых нет?»
Пайплайны калибровки по NPZ исторически ожидали набор файлов `T01_osc.npz`, `T02_osc.npz`, …

В R14 UI умеет **авто‑экспорт** именно в этот формат и именно в стандартную папку:
- `pneumo_solver_ui/workspace/osc/`

## 3) Важные файлы и папки

- `pneumo_solver_ui/pneumo_ui_app.py` — главный UI.
- `pneumo_solver_ui/mech_anim_fallback.py` — fallback‑анимация (2D/3D) на Matplotlib.
- `pneumo_solver_ui/workspace/osc` — осциллограммы NPZ/CSV (в т.ч. экспорт из симуляции).
- `pneumo_solver_ui/workspace/calibration_runs` — результаты pipeline‑ов калибровки/автопилота.
- `pneumo_solver_ui/logs` — логи UI.
- `pneumo_solver_ui/results` — результаты оптимизации.
