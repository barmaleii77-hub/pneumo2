# Diagrammy: принципы диаграмм и навигации (почему так)

## Что считаем “инженерно-быстрым” анализом
1) **Overview → Zoom/Filter → Details-on-demand** (манта Шнайдермана).  
2) **Small multiples** для сигналов одного типа/единиц, **обязательно** с одинаковой шкалой (иначе сравнение нечестное).  
3) Отдельный режим **Δ (delta)**, где базовая линия = 0, а шкала симметрична вокруг нуля.

## Реализация в проекте
- Compare NPZ:
  - базовый режим: overlay
  - режим: Δ to reference
  - playhead (вертикальная линия) для локального “среза” по времени
  - heatmap max|Δ| для “быстрой сортировки: где больше всего отличается”
  - influence (meta→metrics): N×N обзор влияния параметров

- Validation Cockpit:
  - анимация + small multiples ключевых сигналов
  - отчёт по штокам в t0 (цель ≈ 50%)

- Qt Compare Viewer:
  - тот же baseline + lock-scales
  - Δ heatmap (max|Δ|)

## Единицы/нормализации (важно)
- 1 bar = 100000 Pa (и только так), иначе шкалы давления начинают “плыть”.
- Для расходов в даташитах часто используется нормализация (ANR/ISO 8778): отображаем как опцию (display-only).
