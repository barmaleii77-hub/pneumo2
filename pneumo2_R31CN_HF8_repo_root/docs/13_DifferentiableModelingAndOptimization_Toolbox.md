# Fully-differentiable моделирование и оптимизация параметров — toolbox (Python)

Эта заметка — ориентир по библиотекам и практикам для проекта, где цель:
- построить **полностью дифференцируемую** матмодель (пневматика+механика),
- выполнять **оптимизацию параметров** многопараметрической модели,
- делать это максимально устойчиво и воспроизводимо.

---

## 1) Какую «дифференцируемость» реально нужно обеспечить

На практике ломают автодифф:

- `if/else` и `max/min/clip` (разрывы производных или нули градиентов),
- таблицы/lookup без интерполяции,
- «жёсткие» ограничения (обрезки), особенно внутри RHS,
- события/контакт без сглаживания,
- интеграторы без поддержки adjoint/sensitivity.

Стратегия:
1) ввести **smooth-аналоги** переключателей (tanh/sigmoid/softplus),
2) постепенно «ужесточать» smoothness (continuation): от мягкого → к близкому к piecewise,
3) выбрать решатель ODE/DAE, который поддерживает backprop/adjoint.

---

## 2) Решатели ODE, дружественные автодиффу

### JAX: Diffrax
- Diffrax — библиотека решателей ОДУ/СДУ/управляемых ДУ на JAX.
- Поддерживает backprop через `diffeqsolve`, в т.ч. continuous adjoint (BacksolveAdjoint) и другие варианты.

Где смотреть:
- основной док: https://docs.kidger.site/diffrax/
- adjoints: https://docs.kidger.site/diffrax/api/adjoints/

### PyTorch: torchdiffeq (если нужен torch)
- Подходит для нейро-ODE и градиентной оптимизации с torch.
- Минус: часто требует аккуратности со stiff-системами, step-size и т.д.

---

## 3) Оптимизация параметров

### JAX: Optax + (опционально) JAXopt
- Optax — библиотека оптимизаторов (Adam, LBFGS и пр.) для JAX.
- Для больших моделей удобно:
  - Adam/Adagrad для грубого поиска,
  - затем L-BFGS (quasi-Newton) для доводки.

Ссылки:
- Optax docs: https://optax.readthedocs.io/
- GitHub: https://github.com/google-deepmind/optax
- пример L-BFGS: https://optax.readthedocs.io/en/stable/_collections/examples/lbfgs.html
- JAXopt examples: https://jaxopt.github.io/stable/auto_examples/index.html

### CasADi (оптимизация с автодиффом + NLP/OCP)
CasADi — сильный вариант для задач вида:
- оптимизация с ограничениями,
- оптимальное управление,
- DAE/ODE внутри постановки,
- использование IPOPT/и др.

Ссылки:
- Docs: https://web.casadi.org/docs/
- GitHub: https://github.com/casadi/casadi

---

## 4) DAE/жёсткие системы

Если у модели будут:
- алгебраические связи,
- «очень жёсткая» термодинамика/контакт,
то может понадобиться DAE-решатель (например SUNDIALS/IDAS), либо переход к формулировке ODE.

CasADi умеет интегрироваться с внешними численными решателями (в т.ч. через SUNDIALS).

---

## 5) Практики для устойчивой оптимизации «пневматика+механика»

1) **Нормализация параметров**
   - оптимизировать не напрямую площадь/С, а лог-параметры:
     - `logA`, `logC`, `logk`, `loghA`
   - это уменьшает «разброс масштабов».

2) **Штрафы за невалидность**
   - вместо hard-обрезок везде: мягкие барьеры/penalty.
   - пример: `p_abs_min` через smooth-floor.

3) **Стабильный ноль**
   - хранить и тестировать «нулевую позу» (rel0) + автономный selfcheck.
   - это критично: оптимизатор иначе начинает «обманывать» модель.

4) **Continuation по smoothness**
   - начать с более гладкой модели (малые `k_pr`, `k_sign`),
   - затем повышать (модель ближе к реальной piecewise).
