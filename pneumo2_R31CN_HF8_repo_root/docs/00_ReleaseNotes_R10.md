# Obobshchenie R10 — Release Notes

Дата сборки: 2026-01-21

## Что исправлено и улучшено

### 1) Устранён Streamlit‑warning про widget default value + session_state (Graph Studio)
**Симптом (из ваших логов/консоли):**
`The widget with key "gs_cols_..." was created with a default value but also had its value set via the Session State API.`

**Причина:**
В блоке выбора сигналов Graph Studio использовалась комбинация:
- явный `default=...` для `st.multiselect`, и
- прямое присвоение `st.session_state[gs_key] = ...`.

Streamlit считает это конфликтом источников значения.

**Решение в R10:**
- `st.multiselect(..., key=gs_key)` теперь вызывается **без** `default=...`.
- Значение инициализируется в `st.session_state[gs_key]` **до** создания виджета.
- Выбор санитизируется (убираются колонки, которых нет в options).

---

### 2) Fallback‑анимация теперь реально «анимирует»
**Симптом:**
«анимация ничего не анимирует».

**Причина:**
Fallback‑режим был статическим снапшотом по выбранному времени.

**Решение в R10:**
- Добавлены **Play / Pause**, шаг времени и авто‑прогон.
- Используется `streamlit-autorefresh` (если доступен) для периодических rerun.
- Переход по времени делается через **индекс кадра**, что надёжнее float‑слайдера.

Примечание: это fallback‑режим (без custom components), поэтому он устойчивее на разных версиях Streamlit.

---

### 3) Кнопка «Собрать диагностический пакет (zip)»
**Запрос:**
«сделай кнопку скачать пакет для полной диагностики…»

**Решение в R10 (страница Diagnostics):**
- Генерируется единый ZIP (в памяти) для отправки.
- Включает:
  - системную информацию и `pip freeze`;
  - снимок `st.session_state` (без тяжёлых бинарных объектов);
  - `ui_*.log` и `metrics_*.jsonl` (с лимитами по размеру);
  - результаты оптимизации (`results_opt*.csv`, `*_progress.json`, и т. п.);
  - текущие JSON‑ы, которые UI хранит/генерирует (mapping/base/suite и др.).

Также добавлена кнопка **«Stop app (terminate process)»** — закрывает сервер Streamlit, чтобы консоль/процесс не висел после закрытия браузера.

---

### 4) Windows BAT‑скрипты: меньше «кракозябр»
**Симптом:**
«при установке зависимостей в консоль пишутся кракозябры».

**Решение в R10:**
- Скрипты `INSTALL_WINDOWS.bat`, `RUN_WINDOWS.bat`, `RUN_ONECLICK_WINDOWS.bat` переведены на ASCII‑сообщения.
- Добавлен `chcp 65001` и переменные `PYTHONUTF8=1`, `PYTHONIOENCODING=utf-8`.

## Известные ограничения (на сейчас)

1) «Машинка едет по дороге…»
В текущей матмодели дорожный профиль задан как функция времени (`road(t)`), что эквивалентно движению по пространственному профилю при заданной скорости, но **не является полноценной 6‑DOF моделью** (нет явных координат X/Y и т. п.). Это отдельный большой шаг.

2) SVG mapping для пневмосхемы
`default_svg_mapping.json` загружается по умолчанию, но по умолчанию он «пустой» (только schema). Для полноценной подсветки потоков/давлений на схеме нужна привязка (mapping) геометрии SVG к именам узлов/рёбер модели.
