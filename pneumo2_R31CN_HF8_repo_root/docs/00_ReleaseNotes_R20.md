# Release Notes — R20

Дата: 2026‑01‑22

## Главные изменения

### 1) Починен `call_simulate` (критичный баг из логов)
**Проблема:** при расчёте «полного лога по всем тестам» приложение падало с `NameError: name 'call_simulate' is not defined`.

**Решение:**
- Добавлен реальный хелпер `call_simulate(...)` в `pneumo_solver_ui/pneumo_ui_app.py`.
- Хелпер автоматически **фильтрует неподдерживаемые аргументы** (например `max_steps`) по сигнатуре текущей `model.simulate()`.

Эффект:
- «Детальный прогон / полный лог по всем тестам» перестаёт падать.
- Ошибка `simulate() got an unexpected keyword argument 'max_steps'` устранена.

### 2) Логи для Play/Seek в fallback‑анимации теперь реально пишутся
**Проблема:** fallback‑анимация имела логирование, но в UI не передавался callback, поэтому в `ui_*.log` не было событий Play/Speed/Seek.

**Решение:**
- В вызов `render_mech2d_fallback(...)` и `render_mech3d_fallback(...)` передаётся `log_cb=log_event`.

Эффект:
- В логах появляются события: `anim_play`, `anim_pause`, `anim_tick`, `anim_seek`, `anim_speed`.
- Проще диагностировать ощущение «бесконечного расчёта при Play».

### 3) Автоматически создаётся `tests_index.csv` при экспорте synthetic NPZ
**Проблема:** пайплайн калибровки/oneclick/autopilot ожидает `tests_index.csv` с колонкой `имя_теста`.

**Решение:**
- Добавлена функция `write_tests_index_csv(...)`.
- `_ensure_full_npz_for_all_tests()` пишет `tests_index.csv` в `workspace/osc/`.

Эффект:
- oneclick/autopilot корректнее находит тесты/файлы и меньше «ищет несуществующие папки».

### 4) Блок «Флаги (bool)» стал устойчивее
**Проблема:** часть булевых флагов могла быть представлена как `0/1` или как строки `"true/false"` и не попадала в чекбоксы.

**Решение:**
- `_is_bool_like()` считает bool‑подобными также `int 0/1` и строки `"true/false/0/1"`.
- Добавлен корректный парсер `_to_bool(...)`.

Эффект:
- Больше флагов редактируется красиво через чекбоксы, без ручного JSON.

### 5) Батник полной диагностики стал «меню‑ориентированным»
**Проблема:** хотелось запускать диагностику «в 1 клик», без ручного ввода команд и с выбором режима.

**Решение:**
- `RUN_FULL_DIAGNOSTICS_WINDOWS.bat` теперь показывает меню (Minimal/Standard/Full и т.д.).

## Известные ограничения (на этот релиз)
- Кастомный Streamlit‑компонент `mech_anim` может быть недоступен/несовместим с некоторыми версиями Streamlit; по умолчанию используйте fallback‑анимацию.
- Fallback‑Play основан на `st.autorefresh` => при высоком FPS и включённых тяжёлых графиках возможна высокая нагрузка. Используйте режим «скрывать Plotly при Play» или понижайте FPS.

- `RUN_FULL_DIAGNOSTICS_WINDOWS.bat` — меню выбора режима диагностики.
- `docs/02_LogAnalysis.md` — обновлён разбор причин падений (добавлены пункты про `call_simulate`, `tests_index.csv`, логирование анимации).
