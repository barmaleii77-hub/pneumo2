# Release notes — R41 (2026‑01‑24)

Фокус релиза: **стабильность baseline после refresh + расширение наблюдаемости матмодели**
(чтобы KPI/анимация/диагностика опирались строго на результаты расчёта).

## 1) Контекст/требования
- Добавлен свежий контекст: `docs/context/PROJECT_CONTEXT_ANALYSIS.md`.
- Перегенерированы трассируемые требования из `docs/context/*`:
  - `docs/01_RequirementsFromContext_RAW.md`
  - `docs/01_RequirementsFromContext.json`

## 2) Baseline cache: атомарная запись + авто‑восстановление
- Baseline‑кэш (таблица + json‑артефакты) теперь сохраняется **атомарно** (tmp → replace), чтобы при
  перезапуске/refresh не оставались битые файлы.
- Добавлено **автовосстановление** последнего baseline после refresh окна:
  если baseline ещё не посчитан в текущей UI‑сессии, но есть сохранённый указатель
  `cache/baseline/_last_baseline.json`, то baseline подхватывается автоматически.

## 3) Матмодель: расширение наблюдаемости (P0)
В `model_pneumo_v8_energy_audit_vacuum_patched_smooth_all.py` добавлены новые каналы вывода:

- **Рама в точках колёс**: `рама_{ЛП,ПП,ЛЗ,ПЗ}_z_м`, `рама_*_vz_м_с`, `рама_*_az_м_с2`
- **Скорости/ускорения рамы и углов**: `скорость_рамы_z_м_с`, `ускорение_рамы_z_м_с2`,
  `скорость_крен_phi_рад_с`, `ускорение_крен_phi_рад_с2`, `скорость_тангаж_theta_рад_с`,
  `ускорение_тангаж_theta_рад_с2`
- **Колёса**: `скорость_колеса_*`, `ускорение_колеса_*`
- **Относительные перемещения колёс**: `колесо_относительно_рамы_*`, `колесо_относительно_дороги_*`
- **World‑frame (минимум)**: `скорость_vx_м_с`, `путь_x_м`, `yaw_rate_рад_с`, `yaw_рад`, `радиус_поворота_м`
  (пока как постпроцессинг из `ax/ay`, полноценная world‑модель остаётся задачей EPIC‑WORLD).

Важно: ускорения считаются как производные от скоростей (`np.gradient`), то есть это **пост‑обработка**
результатов интегрирования, а не «рисование анимации».

## 5) Дифы/патчи
