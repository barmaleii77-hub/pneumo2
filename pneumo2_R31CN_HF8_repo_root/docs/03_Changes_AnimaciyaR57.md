# Animaciya — UnifiedPneumoApp (R57)

Эта сборка основана на **UnifiedPneumoApp_UNIFIED_v6_22_WINSAFE** и содержит набор правок, ориентированных на **информативную анимацию** и корректный **масштаб/пропорции** (колеса, рама, дорога) в Desktop Animator.

## Что сделано в R57

### 1) Правильный масштаб геометрии во всех 2D-окнах (перед/зад/лево/право)
- Исправлено использование размеров колес:
  - **радиус колеса** и **ширина колеса** теперь берутся из `meta_json` (если присутствует) либо из `pneumo_solver_ui/default_base.json`.
  - больше нет жестко зашитого `wheel_r = 0.18`.
- Исправлена масштабная логика:
  - убран `fitInView()` из каждого кадра (он менял масштаб/зум во времени);
  - 2D окна получают **фиксированную сцену в метрах** + **единый px/м**, автоматически вычисляемый так, чтобы все 2D-виды помещались в окна **одинаковым масштабом**.

### 2) Дорога в каждом виде + нулевой уровень + уровень под каждым колесом
Добавлено во все 2D-виды (передняя ось, задняя ось, левый борт, правый борт):
- линия **z=0 (нулевой уровень дороги)** (пунктир);
- короткие маркеры **уровня дороги под каждым колесом**;
- существующая линия дороги (соединяющая точки под колесами) сохранена.

### 3) Правильное представление колес по виду
- **Перед/Зад (Front/Rear)**: колеса рисуются **прямоугольниками** (ортографический вид спереди/сзади).
- **Лево/Право (Side)**: колеса рисуются **окружностями** (вид сбоку).

### 4) 3D-вид: абсолютный ноль дороги + уровень под колесами
- 3D сцена больше не "сдвигает" Z на среднюю высоту дороги.
  - **Сетка GLGridItem** теперь реально соответствует **z=0**.
- Добавлены:
  - **точки контакта дороги под каждым колесом**;
  - **вертикальные линии** от z=0 до уровня дороги под колесом.

### 5) UI: скроллбар для всего окна (когда не помещается)
Главный cockpit завернут в `QScrollArea`, чтобы на небольших экранах ничего не "пряталось".

### 6) Windows install: зависимости Desktop Animator
Добавлен файл:
- `pneumo_solver_ui/requirements_desktop_animator.txt`

и обновлен:
- `legacy_launchers/INSTALL_WINDOWS.bat`

Теперь он ставит (если файл существует):
- `requirements_desktop_compare.txt`
- `requirements_desktop_animator.txt`

## Как запустить

### Вариант A (через Web UI / Streamlit, рекомендовано)
1) Запустите `legacy_launchers/RUN_WINDOWS.bat`.
2) Откройте страницу `Desktop Animator`.
3) Нажмите экспорт `anim_latest` и/или запустите Desktop Animator из интерфейса.

### Вариант B (напрямую Qt-приложение)
Из корня проекта:
```bat
.venv\Scripts\activate
python pneumo_solver_ui\desktop_animator\app.py
```

Полезные параметры:
- `--no-gl` — отключить OpenGL 3D-вид (если драйвер/GL конфликтует)
- `--npz <path>` — открыть конкретный `.npz`
- `--pointer <path>` — путь к `anim_latest.json`

## Что проверить (быстрый чеклист)
- В Desktop Animator:
  - в окнах **FRONT/REAR** колеса — прямоугольники;
  - в окнах **SIDE L/SIDE R** колеса — окружности;
  - везде есть:
    - пунктирная линия **z=0**;
    - маркеры **уровня дороги под каждым колесом**;
  - масштаб в 2D-видах стабилен во времени.
- В 3D:
  - сетка соответствует **z=0**;
  - видно точки дороги под колесами + вертикальные линии к нулю.
