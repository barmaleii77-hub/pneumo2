# Сглаживание без паразитных офсетов

## Проблема
Классическая “мягкая ReLU” через

`soft_pos(x) = 0.5 * (x + sqrt(x^2 + eps^2))`

даёт **ненулевое значение в нуле**: `soft_pos(0)=eps/2`.

Если использовать её напрямую в силовых законах (контакт, упоры), то появляется **паразитная сила** в точке касания/ограничения. Для статического положения это плохо: система “самоподпирается” даже при нулевой деформации.

## Решение
Вместо `soft_pos(x)` используется “неофсетная” версия:

`pos0(x) = x * d(soft_pos)/dx`

где

`d(soft_pos)/dx = 0.5 * (1 + x/sqrt(x^2 + eps^2))`

Свойства:
- `pos0(0)=0` (нет силы в нуле)
- `pos0(x)~x` при `x >> eps`
- `pos0(x)~0` при `x << -eps`
- гладкая функция

## Где применено
- Контакт шины (`smooth_contacts`):
  - проникновение `pen_in = pen * g_contact`
  - скоростной член только при сжатии и при контакте: `pen_dot_in = (pen_dot * g_vel) * g_contact`
- Упоры (`smooth_stops`):
  - проникновение bump/rebound без `np.maximum` / `np.where`

## Настройка eps
- `smooth_eps_pos_m` и `smooth_eps_vel_mps` — базовые eps
- `smooth_eps_stop_m` и `smooth_eps_stop_vel_mps` — отдельные eps для упоров

Рекомендация: начинать с `1e-4 … 1e-5` (позиция) и `1e-3 … 1e-4` (скорость), затем уменьшать, если нужно более “жёстко”, но следить за численной устойчивостью.
