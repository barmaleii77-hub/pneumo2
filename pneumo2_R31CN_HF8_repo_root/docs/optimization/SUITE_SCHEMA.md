# Suite schema (расширение)

Suite хранится как список словарей в JSON (генерируется UI). Worker превращает её в список тестов.

Файл воркера: `pneumo_solver_ui/opt_worker_v3_margins_energy.py` → `build_test_suite()`.

## Базовые поля

- `включен` (bool) — учитывать тест
- `имя` (str) — имя теста, попадёт в префиксы метрик
- `тип` (str) — шаблон теста (инерция/микро/...)
- `dt` (float) — шаг интегратора для теста
- `t_end` (float) — длительность теста

### Параметры для шаблонов

В зависимости от `тип` используются поля:
- `ay_м_с2`, `ax_м_с2` — для инерционных тестов
- `A_м`, `f_Гц`, `mode`, `bias_м`, `колесо` — для микро/прыжков/комбо

## Новые поля

### `стадия` (int)
Используется StageRunner'ом для фильтрации тестов:
- `0` — быстрые тесты (релевантность/cheap)
- `1` — длинные/дорогие тесты
- `2` — финальные и/или «самые дорогие»

Если поле отсутствует, StageRunner назначит стадию эвристикой.

### `road_surface` (str или JSON)
Включает world-road генерацию профиля дороги внутри модели.

Пример (как строка JSON в UI):

```json
{"type": "iso8608", "class": "C", "seed": 123}
```

Также можно передавать имя пресета, если вы сделали пресеты в `road_surface.py`.

### `road_csv` (str)
Путь до CSV профиля дороги (тайм-серия). Worker в процессе расчёта создаст `road_func`/`road_dfunc`.

Ожидаемые колонки:
- время: `t` / `time` / `время_с`
- дорога (4 колеса): `z_LF,z_RF,z_LR,z_RR` **или** `z0,z1,z2,z3` **или** одиночный `z` (применится ко всем колёсам)

### `axay_csv` (str)
Путь до CSV манёвра. Из CSV строятся `ax_func`/`ay_func`.

Колонки:
- время: `t` / `time` / `время_с`
- ускорения: `ax, ay`

### `params_override` (dict или JSON-строка)
Локальные переопределения параметров модели только для данного теста.

Пример:

```json
{"добавочная_масса_кг": 400, "T_AIR_К": 253.15}
```

Эта механика используется StageRunner'ом для сценариев внешних условий.

## Где смотреть реализацию

- Копирование дополнительных полей: `build_test_suite()`
- Применение `params_override`: `eval_candidate()`
- Тайм-серии CSV: `_compile_timeseries_inputs()` вызывается в `eval_candidate_once()`
