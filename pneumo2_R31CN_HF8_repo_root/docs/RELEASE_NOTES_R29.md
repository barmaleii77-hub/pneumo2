# Release Notes — Obobshchenie R29

Дата: 2026‑01‑23

## Исправления
1. **Детальный прогон (full‑лог)**
   - Исправлен неправильный вызов `call_simulate(...)`, из‑за которого падал детальный расчёт с ошибкой:
     `call_simulate() missing 2 required keyword-only arguments: 'dt' and 't_end'`.
   - Теперь `dt` и `t_end` берутся из выбранного теста (`tests_map`) и сохраняются в `meta` full‑лога.

2. **Нагрузка на GPU/CPU при открытии вкладки «Анимация»**
   - 3D‑canvas (`mech_car3d`) и 2D‑схема (`mech_anim`) переведены в режим *low‑power*:
     - при паузе нет постоянного `requestAnimationFrame`‑рендера;
     - кадр перерисовывается только при изменениях (кадр, взаимодействия мышью/resize) и при редком опросе playhead.

## Изменения поведения
- Авто‑детальный прогон по выбору теста остаётся включён по умолчанию, но теперь не уходит в бесконечный цикл из‑за падения.
- В `baseline_full_cache` добавлены дополнительные поля (`df_drossel`, `df_energy`, `df_atm`, `nodes`, `edges`) — это не ломает старые сценарии, но делает UI богаче при наличии данных.

## Известные ограничения (не решено в R29)
- В матмодели вопрос «используется ли диаметр колёс» требует отдельной проверки/доработки модели (это не чисто UI‑фикс).
- 3D‑камера/масштаб может требовать автоподстройки под экстремальные данные (добавлено в TODO).
