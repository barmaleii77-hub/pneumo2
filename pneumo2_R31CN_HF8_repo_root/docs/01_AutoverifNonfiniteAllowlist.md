# Autoverif: allowlist для non-finite метрик

## Проблема
В метриках оптимизации/диагностики иногда встречаются значения `+inf`, которые **не являются ошибкой физики**.
Классический пример в проекте:
- `t_пересечения_Pmid_с = +inf` означает, что давление в ресивере 3 **не пересекло** `Pmid` в пределах окна теста.

Ранее общий тест «nonfinite» штрафовал любое не конечное число, из-за чего корректные тесты могли получать большой штраф.

## Решение в v6.52
- В `verif_autochecks.py` добавлен allowlist ключей, для которых допускается **только** `+inf`.
- Allowlist можно расширять через `params["autoverif_nonfinite_allowlist"]` (список или строка через запятую).

## Почему это корректно
- Время события (пересечение порога) математически естественно становится `+inf`, если событие не произошло в конечном интервале наблюдения.
- Это не связано с «взрывом» модели (NaN) и не нарушает физику.

## Ограничение
- NaN по-прежнему считается ошибкой.
- Для allowlist разрешается только `+inf` (не `-inf`).
