## 1) Исправления математики

### 1.1 Контакт шины: демпфер должен работать по относительной скорости
**Проблема:** в базе демпфер контакта шины использовал `pen_dot = -zw_dot`.  
Это корректно только если дорога неподвижна (`z_road_dot = 0`).

**Правильно:** при base-excitation контактная «проникновение» описывается:
- `pen = z_road - z_wheel`
- `pen_dot = z_road_dot - z_wheel_dot`

**Что сделано:**
- добавлен расчёт `z_road_dot` (аналитический `road_func_dot` или численно),
- заменён `pen_dot` в расчёте силы контакта.

Файл: `pneumo_solver_ui/model_pneumo_v8_energy_audit_vacuum.py`

### 1.2 Отбойник (bump stop): `delta_dot` нельзя брать как `-s_dot`
**Проблема:** `s_dot` может быть принудительно обнулён в `volumes()` при упоре штока, чтобы «заморозить» объём.
Это ломает демпфирование отбойника.

**Решение:** для отбойника нужна реальная скорость относительного перемещения:
- `delta = z_wheel - z_body`
- `delta_dot = z_wheel_dot - z_body_dot`

Файл: `pneumo_solver_ui/model_pneumo_v8_energy_audit_vacuum.py`

### 1.3 ISO 6358 φ(pr): корректные пределы и согласованность
**Проблема:** реализация в `iso6358_system.py` давала `phi(pr→1) → beta_lam^m`, то есть не стремилась к нулю.
В модели уже была «ламинаризация» к нулю — несогласованность приводила к путанице и потенциально к ошибкам в расчётах эквивалентов.

**Что сделано:**
- синхронизирована формула между `iso6358_system.py` и моделью;
- введены правила:
  - `pr <= 0` → `phi = 1` (вакуум/захлёб),
  - `pr >= 1` → `phi = 0`,
  - `pr >= beta_lam` → линейный хвост к нулю.

Файлы:
- `pneumo_solver_ui/iso6358_system.py`
- `pneumo_solver_ui/model_pneumo_v8_energy_audit_vacuum.py`
