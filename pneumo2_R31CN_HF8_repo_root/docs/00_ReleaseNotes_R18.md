# Release Notes — R18

Дата: 2026‑01‑22

## Главные изменения

### 1) "Бесконечный расчёт" при Play в анимации

Причина: `st.tabs()` **не ленивый** — код внутри *всех* табов выполняется при каждом `rerun`. Когда включалась анимация (auto‑refresh), приложение начинало на каждом тике перерисовывать ещё и тяжёлые разделы (графики/потоки/энерго‑аудит), что визуально выглядело как вечный расчёт.

Решение: заменены ключевые `st.tabs()` на `st.radio()` с ветвлением `if/elif`, чтобы в каждый момент времени выполнялся только выбранный раздел.

### 2) Исправлен кэш детального лога для "все тесты"

В нескольких местах ключ кэша строился как `...::{max_points}::...` без `int()`, из‑за чего при некоторых версиях Streamlit `max_points` становился `1200.0` и ключи переставали совпадать. Это приводило к повторным пересчётам детального лога.

Решение: унифицировано формирование ключа через `int(max_points)`.

### 3) Fallback‑анимации

- `render_mech2d_fallback` и `render_mech3d_fallback` теперь возвращают `(t_selected, corners_selected)`, как и компонентные версии.
- Добавлен параметр `log_cb`, чтобы события Play/Pause попадали в общий UI‑лог.

### 4) Диагностика из интерфейса

Функция `make_ui_diagnostics_zip(...)` расширена: теперь принимает `base_json/suite_json/ranges_json`, умеет включать/исключать каталоги и создаёт `meta.json`. Исправлено несовпадение сигнатуры и вызова.

### 5) Калибровка/Autopilot

Функция `_ensure_full_npz_for_all_tests()` исправлена: теперь корректно генерирует полные логи и экспортирует NPZ (используя реальные выходы `simulate(...)`, а не несуществующие поля).

Дополнительно: при запуске пайплайна в папку результатов делается snapshot UI‑логов (`ui_logs_snapshot/`).

## Совместимость

Релиз ориентирован на Windows 10/11 + Python 3.11+ (проверено также с Python 3.13 у пользователя). Для Streamlit 1.5x учтены изменения API (`width='stretch'`, `on_select`).
