# Внутреннее разбиение шага интегрирования

## Проблема
При внешнем шаге `dt=0.003` и положительном минимальном абсолютном давлении (например 1000 Па) наблюдались:
- разлёт давления,
- уход масс в ограничения и последующий численный "взрыв".

## Решение (6.53)
В модели `model_pneumo_v9_mech_doublewishbone_worldroad.py` добавлено внутреннее подшагивание в блоке интегрирования:
- каждый внешний шаг `dt` разбивается на несколько внутренних `dt_loc`,
- ограничение сверху задаёт параметр `макс_шаг_интегрирования_с`,
- дополнительно контролируется относительное изменение объёма за шаг (`лимит_относит_изменения_объёма_за_шаг`).

Это **не меняет** формат выходных данных (логирование остаётся на внешнем шаге), но существенно повышает устойчивость.

## Рекомендации по настройке
- Стартовое значение: `макс_шаг_интегрирования_с = 0.001`
- Если видите нестабильности (NaN, разлёт давлений) — уменьшайте до `0.0005…0.0003`
- Если нужна скорость и всё стабильно — можно увеличить до `0.0015…0.002`

## Ограничение по объёму
Критерий: `max(|dV|*dt_loc/V) <= лимит_относит_изменения_объёма_за_шаг`.

Это CFL‑подобная страховка для пневмо‑объёмов с быстрым изменением геометрии (ход штоков/рычагов).
