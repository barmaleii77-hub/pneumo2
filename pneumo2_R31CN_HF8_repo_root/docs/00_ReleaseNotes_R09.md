# Obobshchenie R09 — Release Notes

Дата сборки: 2026-01-21

Этот релиз — **инженерный фикс по вашим логам R07** + усиление наблюдаемости приложения (логи/метрики) и устранение “ручных” действий (default mapping).

---

## 1) Исправлено: StreamlitAPIException в fallback‑анимации (2D/3D)

**Симптом (из вашего лога):**

`streamlit.errors.StreamlitAPIException: st.session_state.<key> cannot be modified after the widget with key ... is instantiated`

**Где падало:**
- `pneumo_solver_ui/mech_anim_fallback.py` (`render_mech2d_fallback`, `render_mech3d_fallback`)

**Причина:**
- после создания `st.slider(..., key=...)` код вручную делал `st.session_state[key] = ...`, что Streamlit запрещает.

**Что сделано:**
- убрано ручное присваивание `st.session_state[key]` после создания виджета;
- состояние сохраняется автоматически самим Streamlit.

Результат: **fallback‑2D и fallback‑3D больше не падают** и работают как надёжный режим по умолчанию.

## 2) Исправлено: SyntaxWarning `invalid escape sequence '\\s'`

**Симптом:**
- в консоли при старте предупреждения вида:
  - `SyntaxWarning: invalid escape sequence '\\s'`

**Причина:**
- в `pneumo_ui_app.py` внутри HTML/JS‑строк использовались regex‑литералы JS `split(/\s+/)` без экранирования для Python‑строки.

**Что сделано:**
- заменено на `split(/\\s+/)` в Python‑коде, что даёт корректный `/\s+/` в браузере и убирает предупреждение.

## 3) Добавлено: default mapping JSON + автозагрузка

**Ваше требование:**
- «Нужен mapping JSON — сделай рабочий и грузи по дефолту».

**Что сделано:**
- добавлен файл `pneumo_solver_ui/default_svg_mapping.json` (шаблон mapping v2);
- при старте UI, если mapping не задан, он **автоматически подставляется** в `st.session_state["svg_mapping_text"]`;
- в UI добавлены:
  - отображение источника mapping,
  - кнопка “Сбросить mapping к default”,
  - кнопка “Скачать текущий mapping.json”.

Важно: этот mapping — **шаблон**, чтобы UI всегда запускался без ручных действий. Геометрию веток/узлов вы размечаете встроенными инструментами “Разметка веток/узлов”.

## 4) Добавлено: файловое логирование + метрики процесса

**Ваше требование:**
- «ЛОГИРУЙ ВСЕ! … замеряй время вычислений и загрузку ЦП … можешь сохранять для анализа любую информацию автоматически».

**Что сделано:**
- добавлен единый логгер в `pneumo_ui_app.py`:
  - `pneumo_solver_ui/logs/ui_<session>.log` — JSON‑строки (event + fields)

- при старте пишется событие `ui_start` с версиями и окружением;
- при наличии `psutil` логируются базовые метрики процесса (RSS/VMS, CPU count и т.п.).

Зависимость `psutil` добавлена в `requirements.txt`.
