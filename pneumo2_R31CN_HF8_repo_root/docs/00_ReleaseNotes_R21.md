# Release Notes — R21

Дата: 2026‑01‑22

## Главное

### 1) Исправлен критичный `NameError: parse_sim_output`

В R20 часть пайплайнов падала при попытке сделать **«детальный прогон / полный лог + NPZ (все тесты)»**, потому что в UI использовалась функция `parse_sim_output(...)`, но она не была определена.

В R21 добавлен адаптер `parse_sim_output()` в `pneumo_solver_ui/pneumo_ui_app.py`:

- Нормализует вывод `simulate()` для обоих режимов (`record_full=False/True`).
- Корректно обрабатывает «энерго‑аудит» модель, где `df_Eedges/df_Egroups/df_atm` продублированы в хвосте.
- Не роняет UI, если формат вывода симулятора изменится (возвращает `raw` и пишет событие в лог).

### 2) Восстановлена связка «полный лог → NPZ → oneclick/autopilot»

После фикса `parse_sim_output()` снова работает сценарий:

1) Посчитать **полный лог** для всех тестов (`df_p/df_mdot/df_open`).
2) Сгенерировать **синтетические NPZ** (из расчётных логов) в `workspace/osc/`.
3) Запустить `calibration/pipeline_npz_oneclick_v1.py` и/или `calibration/pipeline_npz_autopilot_v19.py`.

Это важно, потому что реальные замеры (osc) могут появиться позже — а пайплайн уже можно обкатать на расчётных логах.

### 3) Обновлён номер релиза во всех баннерах

- `APP_RELEASE` в UI: **R21**
- Титулы и подсказки в стартовых скриптах: **R21**
