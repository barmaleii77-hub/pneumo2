# 04 — Механо‑энергоаудит

## Зачем это нужно

В пневмо‑подвеске «газ ↔ механика» энергия постоянно перетекает:

- газ совершает работу на поршнях цилиндров (или наоборот — механика сжимает/расширяет газ);
- дорога (как «база» для шины) совершает работу на системе через контактное звено;
- демпферы (в шине и отбойниках) безвозвратно рассеивают мех. энергию;
- инерционные моменты от `ax/ay` совершают работу на крене/тангаже.

Если в модели где-то перепутан знак, неверно назначена площадь поршня, перепутаны CAP/ROD,
или сила приложена не в ту сторону — это почти всегда проявляется «утечкой энергии»:
мех. энергия начинает расти/падать без объяснения внешними работами и диссипациями.

Поэтому в релизе **Matematika63** добавлен «механо‑энергоаудит» как **P0‑инструмент**
для проверки корректности математической модели.

## Что считается

### 1) Механическая энергия системы

Определяем:

- **Кинетика**
  - кузов: `T_body = 0.5*m_body*z_dot^2 + 0.5*I_roll*phi_dot^2 + 0.5*I_pitch*theta_dot^2`
  - колёса: `T_wheels = 0.5*m_w*Σ(zw_dot_i^2)`

- **Потенциал**
  - гравитация: `U_g = m_body*g*z + m_w*g*Σ(zw_i)`
  - пружина (табличная): `U_spr = scale * ∫ F_tab(x) dx`, где `x = x0 + delta`
  - шина: `U_tire = 0.5*k_tire*Σ(pen_pos^2)` (только в контакте)
  - отбойники: `U_stop = 0.5*k_stop*Σ(over_pos^2 + under_pos^2)`

Итого: `E_mech = T_body + T_wheels + U_g + U_spr + U_tire + U_stop`.

### 2) Мощности (внешние + диссипации)

- **Дорога (base excitation)**:
  - `P_road = Σ( F_tire_i * z_road_dot_i )`

- **Газ → механика через цилиндры**:
  - `P_gas = Σ( (p_i - P_atm) * dV_i/dt )` по всем камерам цилиндров
  - это же проверяется независимым способом: `P_gas_force = Σ( F_cyl_corner * s_dot_corner )`

- **Инерционные моменты от ax/ay**:
  - `P_axay = (m_body*ay*h_cg)*phi_dot + (-m_body*ax*h_cg)*theta_dot`

- **Диссипация**
  - шина: `D_tire = c_tire*Σ( pen_dot_pos^2 * g_contact )`  (всегда ≥ 0)
  - отбойники: `D_stop = c_stop*Σ( delta_dot_pos^2*g_over + delta_dot_neg^2*g_under )` (всегда ≥ 0)

### 3) Баланс энергии

Интегрируем (трапециями):

`E_balance(t) = E_mech(0) + ∫(P_road + P_gas + P_axay - D_tire - D_stop) dt`

Ошибка:

- `ΔE = E_mech - E_balance`
- `ΔE_rel = ΔE / max(1, |E_mech|)`

### 4) Проверка p·dV

Считаем:

- `P_pdv_abs = Σ(p_abs * dV/dt)`
- `P_atm_corr = P_atm * Σ(dV/dt)`
- `P_pdv_gauge = P_pdv_abs - P_atm_corr`
- `ΔP_pdv = P_pdv_gauge - P_gas_force`

В идеале `ΔP_pdv → 0` (это проверка правильности мэппинга камер/площадей/знаков).

## Как интерпретировать результаты

- Если `max |ΔE|/|E|` близко к 0 (обычно << 0.1), то:
  - знаки сил/скоростей согласованы,
  - `F*y_dot` для дороги и `p_gauge*dV/dt` для цилиндров реализованы правильно,
  - диссипации не создают энергию.

- Если ошибка большая:
  - проверьте знаки `pen_dot`, `delta_dot`,
  - корректность `CAP/ROD`, площади поршней и формулы `F = (p_cap-Patm)A_cap - (p_rod-Patm)A_rod`,
  - корректность скорости дороги (`z_road_dot`), особенно при «ступеньках/кулачках».
