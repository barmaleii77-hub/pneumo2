# DiagrammyV625Charts — техническая документация изменений

## Зачем это сделано

Для быстрых сравнений результатов симуляции (несколько `.npz`) нужно:

- видеть **одни и те же сигналы** в **одном масштабе**;
- иметь одинаковые базовые функции **в Web и в Qt**: выбор запусков, наложение, Δ‑режим, playhead, блокировка масштаба;
- уметь выставлять **нулевую позу** для перемещений в графиках и анимации (display‑only), чтобы сравнение было интуитивным;
- быстро находить **где именно различия** по сигналам/запускам.

Из‑за этого в v6.25 добавлена веб‑страница сравнения и расширен Qt‑viewer.

## Что добавлено/изменено

### 1) Web: Compare NPZ (Web)
Файл: `pneumo_solver_ui/pages/06_CompareNPZ_Web.py`

Возможности:

- выбор директории с `.npz`, выбор набора запусков и **референса**;
- режимы:
  - **Overlay** — наложение кривых;
  - **Δ to reference** — разница относительно референса (интерполяция на сетку времени референса);
- единицы отображения:
  - перемещения: **mm** или **m**;
  - углы: **deg** или **rad**;
- **Zero baseline** для перемещений и углов:
  - `t0` или медиана на окне `baseline_window_s`;
- **Lock Y**:
  - по сигналу (все запуски в одном диапазоне для данного сигнала);
  - опционально «Lock Y by unit» (единый диапазон для всех выбранных сигналов с одинаковыми единицами);
  - опционально **Symmetric Y** (симметрично относительно 0 — полезно в Δ‑режиме);
- **playhead** (индекс + вертикальная линия на всех графиках);
- **Heatmap max|Δ|**: матрица `run × signal` для быстрой локализации отличий.

### 2) Qt: Compare Viewer — масштабы/нулевая база/единицы
Файл: `pneumo_solver_ui/qt_compare_viewer.py`

Добавлены элементы управления:

- Distance unit: **mm/m**
- Angle unit: **deg/rad**
- Zero baseline (disp/angle) + окно медианы
- Lock Y (per‑signal across runs)
- Lock Y by unit
- Symmetric Y around 0

Также улучшен Δ‑режим:

- референс в Δ‑режиме рисуется как **0‑линия**, остальные — как **Δ‑кривые**.

### 3) Общий фикс: правильные единицы и преобразования
Файл: `pneumo_solver_ui/compare_ui.py`

- исправлен ошибочный пересчёт `*_mm` (раньше умножалось на 1000, что давало микрометры);
- пересчёт Pa→bar делается через `1 bar = 100000 Pa`;
- добавлены опции `dist_unit` и `angle_unit`.

### 4) Web‑совместимость Plotly/Streamlit
Файлы:

- `pneumo_solver_ui/pneumo_ui_app.py`
- `pneumo_solver_ui/param_influence_ui.py`

`st.plotly_chart` и `st.image` имеют разные сигнатуры в разных версиях Streamlit.
Сделано «progressive enhancement»:

- сначала пробуем `width="stretch"`;
- если не поддерживается — падаем на `use_container_width=True`;
- если и это не поддерживается — вызываем базовую версию без параметров.

### 5) Анимации: нулевая поза и «середина хода»
Файлы:

- `pneumo_solver_ui/components/mech_anim/index.html`
- `pneumo_solver_ui/components/mech_car3d/index.html`

- текст по штокам дополняется **% хода** и подсветкой близости к 50%.

## Алгоритмы

### Нулевой базис (baseline)

1. Берём сигнал `y(t)` в выбранных единицах.
2. Если `baseline_window_s > 0`, вычисляем `baseline = median(y[t<=t0+window])`.
3. Иначе `baseline = y[t0]`.
4. Отображаем `y' = y - baseline`.

По умолчанию baseline применяется к единицам `m/mm/deg/rad`.

### Δ‑режим

1. Выбирается референс‑запуск.
2. Для каждого другого запуска интерполируем `y_i(t)` на сетку времени `t_ref`.
3. Показываем `Δy(t_ref) = y_i(t_ref) - y_ref(t_ref)`.

### Lock Y

1. Для каждого сигнала собираем все отображаемые серии (overlay или Δ).
2. Находим `ymin/ymax` по конечным значениям.
3. Если включён `Symmetric Y`: `m=max(|ymin|,|ymax|)` и диапазон `[-m, +m]`.
4. Применяем диапазон к plot.

`Lock Y by unit` делает то же, но сначала агрегирует диапазон по единицам.

## Известные ограничения

- Нулевая поза/середина хода сейчас **display‑only** (визуализация и контроль). Автоматический расчёт статической позы/подбор начальных условий — отдельная задача из TODO.
- Lock Y by unit не объединяет разные физические единицы (это бессмысленно).
