# PneumoApp_v6_80_R176_WINDOWS_CLEAN_R15 — release notes

Дата: 2026-03-18

## Что исправлено

### Solver / solver-points
- Исправлен service-only вывод верхнего рычага (`arm2_*`):
  - `arm2_joint` больше не совпадает с `arm_joint`.
  - Верхний рычаг строится как отдельный сегмент, параллельный нижнему рычагу в локальной плоскости YZ.
- Исправлена привязка `cyl2_bot`:
  - теперь нижняя точка `C2` интерполируется по линии второго рычага,
  - а не по нижнему рычагу с тем же outboard-joint.

### Диагностика
- `suspension_geometry_diagnostics` теперь ловит отдельный класс ошибки:
  - коллапс `arm_joint == arm2_joint`.
- `inspect_suspension_geometry.py` теперь печатает `arm_joint_delta_max` и флаг `coincident_arms`.
- `desktop_animator/self_checks.py` теперь предупреждает, если верхний и нижний рычаги сходятся в одну точку.

### Тесты
- Добавлен регрессионный тест на коллапс upper/lower arm joints.
- Усилен smoke на active generators:
  - проверяется, что `arm2_joint_z != arm_joint_z`;
  - проверяется, что `cyl2_bot` лежит на линии второго рычага.

## Что не делалось сознательно
- Не вводились новые raw/base параметры.
- Не менялись по умолчанию значения `верх_Ц1_*` / `верх_Ц2_*` и `низ_Ц1_*` / `низ_Ц2_*`.
- Не рисовалась фальшивая геометрия цилиндров в аниматоре.

## Честный статус
- Последний пользовательский bundle `SEND_20260318_113445_auto-exit_bundle.zip` уже содержит 2 рычага и 2 канала цилиндров, но всё ещё показывает коллапс `arm2_joint == arm_joint`.
- Патч R15 закрывает это на стороне solver-points contract.
- Для полного монтажного разведения `C1/C2` по верхним точкам по-прежнему нужны разные raw/base значения параметров `верх_Ц1_*` и `верх_Ц2_*`.
