# Release Notes — R19

Дата: 2026‑01‑22

## Исправления (по последним логам и твоим сообщениям)

### 1) Ошибка `simulate() got an unexpected keyword argument 'max_steps'`
- В `pneumo_ui_app.py` добавлен адаптер `call_simulate()`:
  - если текущая версия `simulate()` **не принимает** `max_steps` (или любой другой kwarg) — он **игнорируется**, приложение не падает;
  - адаптер фильтрует kwargs по сигнатуре `simulate()` (через `inspect.signature`).

### 2) «Полный лог по всем тестам» больше не ломается из‑за несовпадения формата возвращаемых данных
- Раньше код пытался распаковать 7 значений, а модель (energy_audit) возвращает длинный tuple.
- Теперь используется `parse_sim_output()`, который нормализует разные варианты возвращаемых значений:
  - короткий формат (legacy),
  - формат energy_audit (`df_main`, `df_energy_edges/groups/atm`, `df_p`, `df_mdot`, `df_open`),
  - 7‑колоночный (если появится в других версиях).

### 3) Автоматический экспорт синтетических NPZ по всем тестам («oneclick: full logs + NPZ») реально пишет *полный* лог
- `_ensure_full_npz_for_all_tests()` переписан так, чтобы:
  - запускать модель с `record_full=True`;
  - брать `dt`/`t_end` из тест‑сьюта (`baseline_tests_map`), а не использовать дефолты модели;
  - сохранять в кэш кадры `df_main/df_p/df_q(df_mdot)/df_open` и экспортировать NPZ.

### 4) Fallback‑анимация стала понятнее
- В 2D‑fallback добавлена строка статуса: оценка **крен φ** и **тангаж θ** (в градусах) + пояснение знаков:
  - `φ>0` — левый борт вверх;
  - `θ>0` — нос вниз (это соответствует текущим знакам в модели).

### 5) Убран спам‑warning `use_container_width` (Streamlit deprecation)
- В `mech_anim_fallback.py` `st.pyplot(... use_container_width=True)` заменён на новый API `width='stretch'` с fallback на старый.

### 6) «Play = бесконечный расчёт» стало контролируемо
- Проблема: fallback‑анимация использует авто‑обновление → каждый кадр = rerun всего приложения.
- Решение: добавлен «предохранитель производительности» — пока fallback Play активен, функция `plot_lines()` временно **не строит** тяжёлые Plotly‑графики.
- Управление: в сайдбаре чекбокс **«При Play (fallback) скрывать Plotly-графики»** (по умолчанию включён).

## Совместимость

- Релиз рассчитан на ту же структуру проекта, что и R18.
- Если у тебя свой `model_*.py` с другой сигнатурой `simulate()` — UI должен пережить это благодаря адаптеру.
