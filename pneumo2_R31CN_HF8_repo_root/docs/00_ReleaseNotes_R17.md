1) Починить падение UI на новых версиях Streamlit из-за `on_select=None` в `st.plotly_chart`.

## Главное

### 1) Фикс критического падения: Plotly + Streamlit `on_select=None`

Ошибка из лога:
- `StreamlitAPIException: You have passed None to on_select`.

Причина:
- в новых Streamlit `on_select` не принимает `None` (допустимы только `'ignore'`, `'rerun'` или callable), поэтому передача `None` приводит к исключению.

Что сделано:
- `safe_plotly_chart()` теперь **не передаёт `on_select` в Streamlit, если он равен `None`**, и добавлен многоступенчатый fallback по сигнатурам (`width='stretch'` → `use_container_width=True` → самый старый режим).

Файл:
- `pneumo_solver_ui/pneumo_ui_app.py` → `safe_plotly_chart()`.

### 2) Анимация: явный индикатор backend + логирование выбора

Чтобы разбирать кейсы вида «Play ушёл в бесконечный пересчёт»:
- в блоке анимации добавлено логирование события выбора backend:
  - `anim_backend_selected` (`backend=component|fallback`, `dataset_id=...`).
- в UI-описании исправлена неоднозначность: теперь явно написано, что **по умолчанию выбран fallback (matplotlib)**.

Важно:
- в компонентном режиме Play идёт **в браузере** (сервер не делает тяжёлых вычислений на каждом кадре),
- в fallback режиме Play использует `st_autorefresh` и вызывает rerun всего приложения с заданным FPS (это может выглядеть как «вечный расчёт», если рендер тяжёлый).

Файлы:
- `pneumo_solver_ui/pneumo_ui_app.py`
- `pneumo_solver_ui/mech_anim_fallback.py` (логирование Play/Pause уже было)

### 3) «Режимы и флаги»: все bool — чекбоксы (усиление типизации)

Добавлен «якорь типов»:
- список `BASE_BOOL_KEYS` строится сразу после загрузки `default_base.json`.
- дальше UI показывает чекбоксы для этих ключей даже если значения где-то «уплыли» в 0/1 или `numpy.bool_`.

Файл:
- `pneumo_solver_ui/pneumo_ui_app.py`.
