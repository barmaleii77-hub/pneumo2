# Release Notes — R26

Дата: 2026‑01‑22

## Главное

### 1) Baseline больше не нужно пересчитывать после refresh окна браузера
Добавлен **дисковый кэш baseline**.  
При открытии приложения (новая Streamlit‑сессия) UI автоматически пытается загрузить baseline из:

`pneumo_solver_ui/workspace/cache/baseline/<key>/baseline_table.csv`

Ключ `<key>` зависит от:
- текущего `base_override` (параметры),
- текущего набора тестов (suite: тесты + dt + t_end + targets),
- выбранного файла модели.

### 2) Детальные прогоны (full‑лог) тоже кэшируются на диск
Это устраняет повторный тяжёлый расчёт при «play/перерисовках» и при обновлении страницы.

Файлы кэша детального прогона:

`pneumo_solver_ui/workspace/cache/baseline/<key>/detail/<test>__mp<max_points>__full<0|1>.pkl.gz`

### 3) Авто‑детальный прогон теперь по умолчанию OFF
Чекбокс **«Авто‑расчёт при выборе теста»** теперь выключен по умолчанию, чтобы исключить ситуацию:
- baseline закончен,
- индикатор «готово»,
- а потом внезапно стартует тяжёлый детальный прогон и грузит CPU.

Если нужен автоподсчёт — включите чекбокс.

## Логирование

Добавлены события:
- `baseline_loaded_cache`
- `baseline_saved_cache`
- `detail_loaded_cache`
- `detail_saved_cache`

Их можно найти в:
- `pneumo_solver_ui/logs/ui_combined.log`
- `pneumo_solver_ui/logs/metrics_combined.jsonl`

## Совместимость

Кэш привязан к параметрам/тест‑сету/файлу модели.  
Если вы поменяли параметры или suite — будет другой `<key>`, и кэш не подхватится (это нормально).
