# rel0 (нулевая поза) и автономная самопроверка кинематики

## Зачем

В проекте **обязательно** нужны два свойства:

1. **Единая нулевая поза** для всех перемещений в графиках и анимации:
   - дорога горизонтальна (0 м);
   - подвеска в **статическом равновесии** на ровной дороге;
   - штоки/пружины находятся в положении, которое следует из статического расчёта и параметров пневмосистемы
     (целевое — около **середины хода**).

2. **Автономная проверка**, что все взаимные перемещения согласованы:
   - колесо ↔ дорога;
   - колесо ↔ рама (угол);
   - шток ↔ геометрия подвески (через δ = z_wheel - z_frame_corner);
   - нулевая база rel0 действительно равна 0 на t=0.

Если хотя бы один из этих пунктов не выполняется, результаты могут выглядеть «правдоподобно» в одном графике,
но будут физически некорректны при анализе, оптимизации и особенно в анимации.

---

## rel0: что это и где реализовано

**rel0** — это добавочные колонки вида `*_rel0`, которые равны **сигналу минус его значение на t=0**.

Пример:
- `колесо_ЛП_z_м_rel0 = колесо_ЛП_z_м - колесо_ЛП_z_м(t=0)`

Это гарантирует, что **в нулевой позе** (ровная дорога, статика, t=0) любые смещения на графиках начинаются с нуля.

Где реализовано:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`
  - флаг `use_rel0` (по умолчанию `True`) добавляет `*_rel0` для всех численных колонок `df_main`.

## Как rel0 используется в UI и анимации

### UI (Streamlit)
- В `pneumo_solver_ui/pneumo_ui_app.py` графики по умолчанию **предпочитают** `*_rel0`,
  сохраняя исходные имена сигналов в легенде/селекторе.
- Флаг в интерфейсе: `use_rel0_for_plots` (по умолчанию `True`).

### Desktop Animator / mech_anim
- **LAW (ABSOLUTE-ONLY ANIMATION):** Desktop Animator **никогда** не подставляет `*_rel0` автоматически.
  Он рисует геометрию строго по **абсолютным** сигналам, экспортированным моделью.
  Это исключает «фантазирование» базиса/клиренса/высоты кузова.
- `*_rel0` остаются как удобный формат для графиков/анализа. Если в Animator нужно показать rel0
  (например, как числовой оверлей), это делается **явно**: через `get_rel0()` или по имени колонки.
- `components/mech_anim/index.html` может показывать штоки в виде `Δ / abs` (смещение от нуля / абсолют),
  но сама геометрия/координаты для отрисовки берутся из **абсолютных** сигналов модели.

## Автономная самопроверка: что именно проверяется

В конце каждой симуляции (и частично на t=0) выполняется проверка,
результат которой пишется в `df_atm` в колонках `mech_selfcheck_*`.

Файл:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`

Ключевые проверки:

1. **Тождества колесо ↔ рама и колесо ↔ дорога**  
   Для каждого угла `c ∈ {ЛП,ПП,ЛЗ,ПЗ}` проверяется:
   - `(z_wheel_center - z_frame_corner) - (колесо_относительно_рамы_c)` ≈ 0
   - `(z_wheel_center - z_road) - (колесо_относительно_дороги_c)` ≈ 0

2. **Согласованность кинематики штока**  
   Проверяется, что логируемый ход штока `положение_штока_Цi_c_м` соответствует пересчёту из δ и геометрии.
   Это ловит типовые ошибки «перепутали угол/шток/знак».

3. **rel0-база**  
   На t=0 все `*_rel0` должны быть ~0.

4. **Статика на t=0**  
   При `ax=0, ay=0` и ровной дороге ожидаем:
   - `z̈_body(t0) ≈ 0`
   - `z̈_wheels(t0) ≈ 0`

5. **Эвристики для ax/ay (mapping sanity)**
   - если `ax != 0`, а перед/зад двигаются одинаково → подозрение на перепутанную геометрию
   - если `ay != 0`, а лев/прав двигаются одинаково → подозрение на перепутанную раскладку углов

## Управление и режим «строгости»

Параметры:
- `mechanics_selfcheck` (по умолчанию `True`)
- `mechanics_selfcheck_tol_m` (по умолчанию `1e-6` м)
- `mechanics_selfcheck_strict` (по умолчанию `False`)

Если `mechanics_selfcheck_strict=True`, то при превышении допуска модель выбросит `RuntimeError`.
Это удобно для оптимизации параметров: «плохие» точки можно сразу считать невалидными.

---

## Какие метрики смотреть (df_atm)

Минимальный набор:
- `mech_selfcheck_ok` (0/1)
- `mech_selfcheck_err_wheel_frame_m`
- `mech_selfcheck_err_wheel_road_m`
- `mech_selfcheck_err_stroke_C1_m`, `mech_selfcheck_err_stroke_C2_m`
- `mech_selfcheck_rel0_t0_maxabs`
- `mech_selfcheck_t0_static_checked` (0/1)
- `mech_selfcheck_t0_z_ddot_m_s2`
- `mech_selfcheck_t0_wheel_ddot_max_m_s2`
- `mech_selfcheck_msg` (строка с причинами/подсказками)
