# Diagrammy Charts Guide v6.39

Этот документ описывает, **какие типы диаграмм** доступны в Compare (Web/GUI), зачем они нужны и какие принципы используются, чтобы **подписи/оси/легенды не накладывались**.

## 1) Базовые диаграммы времени

### 1.1 Overlay (t → y)
- **Что показывает:** несколько прогонов на одной шкале для одного сигнала.
- **Зачем:** быстро оценить разброс и влияние параметров на форму сигнала.
- **Навигация:** zoom/pan, playhead (вертикальная линия) и синхронизация по времени.

### 1.2 Delta (t → Δ)
- **Что показывает:** разницу между прогоном и опорным (reference) для каждого сигнала.
- **Зачем:** «что изменилось» понятнее, чем «какая абсолютная форма».
- **По умолчанию:** шкала симметрична вокруг 0 и одинаковая для сопоставимых сигналов.

## 2) Матрицы/теплокарты

### 2.1 Max |Δ|: сигнал × прогон
- **Что показывает:** максимум абсолютной разницы (по времени) относительно reference.
- **Зачем:** быстро найти «какой прогон больше всего отличается» и «по каким сигналам».

### 2.2 Δ(t): сигнал × время
- **Что показывает:** эволюцию Δ во времени одновременно по нескольким сигналам.
- **Зачем:** увидеть «когда» происходят отличия и в каком режиме.

## 3) Статистические и «распределенческие» графики

### 3.1 Histogram / ECDF
- **Зачем:** сравнение не только по экстремумам, но и по распределению значений/ошибок.
- ECDF (эмпирическая функция распределения) обычно читается лучше, чем 10 гистограмм.

### 3.2 Density contour (2D)
- **Что показывает:** плотность точек (x,y) во времени.
- **Зачем:** «облако состояний» (например, давление‑расход или ход‑крен).

## 4) Частотные и фазовые

### 4.1 PSD (Welch)
- **Что показывает:** спектр мощности (приближённо «какие частоты доминируют»).
- **Зачем:** вибрации/резонансы/качество демпфирования.

### 4.2 Phase plot (y1 vs y2)
- **Что показывает:** фазовую траекторию.
- **Зачем:** устойчивость, цикл, насыщение, нелинейности.

## 5) N→N анализ влияния параметров

В Compare (Web) доступен блок «Influence / Correlation»:
- из мета‑параметров (meta_json → params) строятся признаки;
- из сигналов строятся метрики (амплитуда, max|Δ|, RMS Δ и т.п.);
- считаются корреляции/ранжирование.

Это покрывает задачу: **изменение N параметров → изменения N метрик**.

## 6) Политика «нет наложений текста»

Чтобы подписи/подсказки/легенды не мешали восприятию:

1) **Сокращение длинных названий**
- отображаются короткие подписи;
- полное имя доступно через hover или отдельные списки.

2) **Разрежение tick‑подписей на категориальных осях**
- для heatmap / radar / parallel координат показывается не каждая подпись.

3) **Авто‑margins и standoff для осей (Plotly)**
- включён `automargin=True`, увеличены поля при длинных названиях.

4) **Легенда вне графика**
- легенда переносится вправо/вверх, чтобы не перекрывать данные.

5) **GUI (PyQtGraph): autoExpandTextSpace**
- оси расширяют область под текст;
- при необходимости подписи заменяются индексами + расшифровка в панели.
