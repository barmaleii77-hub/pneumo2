# Release Notes — R25

Дата: 2026‑01‑22

## Главное

### 1) Suite/UI тесты начали реально воздействовать на модель (ax/ay/дорога)
**Исправлено системное несоответствие форматов:**
- UI/`default_suite.json` задаёт тесты через поля `тип`, `ay/ax`, `A/f`, параметры кочки и т.п.
- Модель же исторически ожидала готовые функции `ax_func/ay_func/road_func`.

В R25 в файл модели добавлен компилятор suite‑тестов:
- если в тесте есть `тип` и не переданы явные `ax_func/ay_func/road_func`,
  модель автоматически строит эти функции и дальше считает как обычно.

Поддержаны типы:
- `инерция_крен` (ступенчатое `ay`, с плавным включением)
- `инерция_тангаж` (ступенчатое `ax`, с плавным включением)
- `микро_синфаза`
- `микро_разнофаза`
- `кочка_одно_колесо`
- `кочка_диагональ` (с задержками по геометрии/скорости/углу — приближение)
- `комбо_крен_плюс_микро`

**Побочный полезный эффект:** 2D/3D анимация перестала выглядеть «сломано», потому что
временные ряды перестали быть нулевыми.

---

### 2) Анимация: устранён «ложный эффект бесконечного расчёта»
Причина жалоб типа «на Play уходит в бесконечный расчёт/грузит CPU» обычно была такая:
- при fallback‑анимации (matplotlib) Play реализован через `streamlit_autorefresh`, что приводит к частым rerun,
  а значит — к частому пересозданию изображений и нагрузке на CPU.

В R25:
- режим Component (2D/3D) оставлен основным (Play в браузере, без server‑rerun на каждый кадр),
- fallback сохранён только как резервный вариант (см. `docs/03_Troubleshooting.md`).
