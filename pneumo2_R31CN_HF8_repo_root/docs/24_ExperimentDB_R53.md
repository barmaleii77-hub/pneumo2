# ExperimentDB (DuckDB/SQLite) — R53

## Зачем
- глобальный дедуп/кэш: не считать одинаковый набор параметров дважды,
- воспроизводимость: все trial'ы и конфиги остаются в одном файле,
- аналитика: HV во времени, вклад источников, причины ошибок.

## Таблицы
- `runs`: манифест запуска
- `trials`: PENDING/RUNNING/DONE/ERROR
- `cache`: глобальный кэш DONE результатов по (problem_hash, param_hash)
- `run_metrics`: HV прогресс

## Рекомендация по конкуренции
Для embedded файловых DB предпочтительна схема **single-writer** (один писатель) + много читателей.
