# Release notes — ObobshchenieR12

## Главное

- **Исправлен критический крэш UI**: `NameError: decimate_n is not defined` в *Graph Studio*.
- **Диагностика стала устойчивее**:
  - страница **Diagnostics** больше не «умирает» из‑за `CalledProcessError` — stdout/stderr сохраняются даже при ненулевом коде возврата.
  - one‑click скрипт **RUN_FULL_DIAGNOSTICS_WINDOWS.bat** расширен: теперь делает smoke‑проверку UI + «оптимизацию‑smoke» и пишет снимки CPU/RAM до/после каждого шага.
- **Версионирование**: в логах/интерфейсе версия теперь **R12** (раньше местами оставалось `R10`).

## Что именно исправлено

### 1) Крэш Graph Studio (NameError)
**Симптом:** после успешного расчёта теста приложение падало на строке сборки ключа `gs_key`.

**Причина:** в строке ключа использовались переменные `decimate_n` и `units_auto`, которые не были определены в этой области.

**Фикс:** `gs_key` теперь строится только из определённых переменных (стабильно привязан к текущему набору данных/источнику).

### 2) Diagnostics: сохранение вывода при ошибках
**Симптом:** при ненулевом возврате subprocess появлялся `CalledProcessError`, и пользователю было не видно stdout/stderr.

**Фикс:** запуск subprocess переведён на `subprocess.run(..., capture_output=True, check=False)` и вывод сохраняется всегда.

### 3) Полная диагностика одним кликом стала «по‑настоящему комплексной»
Скрипт `pneumo_solver_ui/tools/run_full_diagnostics.py` теперь дополнительно:
- запускает headless UI smoke‑test (Streamlit) и сохраняет лог/HTML;
- запускает «оптимизацию‑smoke» уже в режиме `standard`;
- пишет снимки CPU/RAM (если доступен `psutil`) до/после каждого subprocess‑шага.

## Что отправлять мне при проблемах

1) Запустите **RUN_FULL_DIAGNOSTICS_WINDOWS.bat**.
2) Возьмите архив вида `RUN_YYYYMMDD_HHMMSS_full_diagnostics_standard.zip`.
3) Пришлите архив сюда.

Альтернатива (если UI открывается): на странице **Diagnostics** нажмите **Собрать ZIP‑пакет диагностики** и пришлите получившийся ZIP.
