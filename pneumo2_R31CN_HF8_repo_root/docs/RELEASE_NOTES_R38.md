# Release Notes — Obobshchenie R38

Дата: 2026‑01‑24

Этот релиз — **про консолидацию требований из контекста** и про **стабильность анимации**
(без “фокусов” и без лишней нагрузки при простом заходе на вкладку).

## 1) Требования из контекста архива (полная трассируемость)

Добавлено:

- `docs/context/` — сохранённые файлы контекста (TXT/MHTML) из архива пользователя.
- `docs/01_RequirementsFromContext_RAW.md` — **автоматически извлечённый** список строк‑требований/ожиданий.
- `docs/01_RequirementsFromContext.json` — тот же список в машиночитаемом формате.
- `pneumo_solver_ui/tools/extract_requirements_from_context.py` — скрипт, который пересобирает RAW+JSON из `docs/context/`.

Зачем:

- Чтобы больше не терять требования в переписке.
- Чтобы можно было привязать каждый пункт к реализации/ TODO.

## 2) Анимация: защита от “рассинхрона” при refresh

Проблема, которую лечим:

- Вкладка анимации синхронизируется через localStorage. Если **после refresh**
  `dataset_id` совпал со старым (тот же `cache_key`), то браузер мог успеть
  кратковременно “подхватить” старое состояние playhead (включая `playing=true`).
  Это проявлялось как лишняя нагрузка и “дергание” ползунка.

Исправление:

- `dataset_id` для UI‑компонентов теперь включает **nonce сессии** (`get_ui_nonce()`),
  поэтому для браузера это всегда “новый датасет” после refresh.

## 3) Снижение GPU/CPU нагрузки в паузе

- В 2D и 3D компонентах увеличена частота “пауза‑poll” (реже перерисовка в паузе).
  Это уменьшает загрузку GPU, когда пользователь просто открыл вкладку.

Файлы:

- `pneumo_solver_ui/components/mech_anim/index.html`
- `pneumo_solver_ui/components/mech_car3d/index.html`

## 4) Документация

- `docs/01_RequirementsFromContext.md` обновлён: теперь ссылается на новый `docs/context/` и RAW/JSON.

---

### Что НЕ делалось намеренно

- Не добавлялась “поддельная физика” в анимацию. Анимация по‑прежнему рисует только
  то, что пришло из расчёта (и если чего‑то нет — показывает 0/пусто и/или предупреждает).
