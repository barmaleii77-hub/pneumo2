# Release notes — ObobshchenieR16

Дата: 2026-01-22

## Цель релиза

Сделать поведение “из коробки” более надёжным на Windows/Streamlit, убрать типовые зависания/ошибки на анимации и сделать **калибровку/Autopilot** более понятными и “one‑click”, даже если реальных замеров ещё нет.

## Главное

### 1) Анимация: дефолт → fallback (matplotlib)

- В UI **движок анимации по умолчанию** теперь **«Встроенный (matplotlib, надёжно)»**.
- Компонентный режим оставлен как экспериментальный (можно включать вручную), но больше не мешает запуску “по умолчанию”.

Почему: в некоторых окружениях Streamlit‑компоненты (iframe + postMessage протокол) могут ломаться/вести себя непредсказуемо, а fallback гарантированно работает.

### 2) 3D fallback: видим «дорогу»

- В 3D fallback добавлены:
  - чекбокс **«Показывать трассу дороги (след колёс)»**
  - слайдер **«Окно трассы (сек)»**
- Рисуется траектория точек контакта колёс с дорогой в мировых координатах вокруг текущего времени — это визуально отвечает на вопрос «почему колёса двигаются, а дорога нет».

### 3) Калибровка: нормальный выбор файлов NPZ/CSV

В expander “Калибровка и Autopilot (NPZ/CSV)” добавлен блок:

- **Mapping файлов ➜ Txx_osc.npz**
  - можно выбрать любой файл NPZ/CSV из `workspace/osc/`
  - одним кликом создать/обновить ожидаемые `T01_osc.npz`, `T02_osc.npz`, …
  - mapping сохраняется в `workspace/osc/mapping_tests_files.json`

Это снимает путаницу с “autopilot ищет папку/файлы которых нет” и не заставляет переименовывать руками.

### 4) One‑click пайплайны без консоли

В калибровке добавлены кнопки:

1) **Полный лог + NPZ (все тесты)** — генерирует расчётные NPZ из текущего baseline (для самопроверки форматов)
2) **… ➜ oneclick** — после подготовки NPZ запускает `pipeline_npz_oneclick_v1.py`
3) **… ➜ autopilot (minimal)** — после подготовки NPZ запускает `pipeline_npz_autopilot_v19.py`

Важно: без реальных измерений калибровка “по смыслу” мало полезна, но пайплайн можно гонять как тест связки форматов/скриптов.

### 5) Диагностика: фикс падения run_full_diagnostics

- `tools/run_full_diagnostics.py` теперь гарантированно создаёт папку `reports/` перед записью stdout/stderr, чтобы не падать на Windows.
