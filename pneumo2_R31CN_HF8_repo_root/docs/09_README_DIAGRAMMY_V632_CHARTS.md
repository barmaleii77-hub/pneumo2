# Диаграммы V6.32: Comparative Charts Suite (Web + Windows GUI)

Этот документ описывает, **какие диаграммы мы выводим, почему именно так** и как ими пользоваться для анализа результатов симуляций физических процессов (подвеска + пневмосистема).

---

## 1) Почему именно такой набор диаграмм

В задачах сравнения прогонов есть две типовые ловушки:

1) **Сравнение «на глаз» на разных шкалах** (даже при одинаковых данных) — вносит систематические ошибки.
2) **Поиск причины по десяткам сигналов** занимает много времени, если нет “summary views” (тепловых карт/метрик).

Поэтому набор построен по принципу:

- **Small multiples** (много одинаковых графиков, один под другим, общий X) для детального анализа.
- **Coordinated views**: линии + теплокарта + метрики + playhead/окно времени.
- **Одни и те же принципы и опции** в Web и в Desktop, но в Desktop возможности навигации богаче.

## 2) Web: Compare NPZ

### 2.1 Overlay vs Δ (Delta)

- **Overlay** — наложение сигналов нескольких прогонов.
- **Δ (Delta)** — разница каждого прогона к выбранному reference.

Преимущества Δ:
- сразу видно, что изменилось;
- можно включать **симметричную шкалу** по Y, чтобы «плюс» и «минус» читались одинаково.

### 2.2 Единый ноль (static baseline)

Опция *Zero baseline* делает «нулевую позицию» общей:
- дорога горизонтальна,
- подвеска в статике,
- перемещения/углы показываются относительно статического положения.

Это **display-only** операция (данные не портятся), работает одинаково и в Web, и в GUI.

### 2.3 Одинаковые масштабы

Критично для корректного сравнения:
- *Lock Y by signal* — один и тот же сигнал во всех прогонах имеет одинаковый диапазон;
- *Lock Y by unit* — все сигналы в одинаковых единицах (например, mm) приводятся к единому диапазону.

### 2.4 Теплокарта Run × Signal

Теплокарта строит матрицу **max|Δ|** по выбранным сигналам и прогонам.
Это “summary view”: быстро показывает, где самые большие расхождения.

### 2.5 Теплокарта Time × Signal (для одного прогона)

Дополнительный слой диагностики: для выбранного прогона показывает **когда** именно возникло расхождение и в каких каналах.

### 2.6 Influence (N параметров → N метрик)

Блок *Influence* строит корреляционную теплокарту:
- строки: числовые параметры из meta-json прогона,
- столбцы: метрики сигналов (RMS, и др.),
- значения: корреляции.

Это даёт быстрый ответ “что на что влияет” без ручного перебора.

## 3) Desktop (Windows GUI): Qt Compare Viewer

GUI делает то же самое, что Web, но быстрее и удобнее:

- мгновенный скроллинг/зум на pyqtgraph,
- общий playhead и окно времени,
- теплокарта Run × Signal с кликом по ячейке,
- теплокарта Time × Signal,
- отдельный таймлайн клапанов/событий.

Рекомендуемый workflow:

1) Выбрать референсный прогон.
2) Перейти в Δ режим.
3) Открыть Run×Signal heatmap и найти топ-каналы.
4) Кликнуть по каналу → перейти на линию → уточнить участок по навигатору.
5) Открыть Time×Signal heatmap, чтобы увидеть «когда».
6) Если нужно: перейти в Influence и посмотреть какие параметры коррелируют с метриками.

## 4) Где лежит код

- `pneumo_solver_ui/compare_ui.py` — ядро утилит (unit inference, baseline, lock ranges, heatmap builders).
- `pneumo_solver_ui/compare_npz_web.py` — Web-страница Compare NPZ.
- `pneumo_solver_ui/validation_cockpit_web.py` — Web validation cockpit.
- `pneumo_solver_ui/qt_compare_viewer.py` — Desktop/Windows Compare Viewer.
- `pneumo_solver_ui/pages/06_CompareNPZ_Web.py` — тонкий wrapper, зовёт `render_compare_npz_web()`.
