# Release Notes — R15

Дата сборки: 2026‑01‑22

## 1) Ключевые изменения

### 1.1 Исправлен эффект «бесконечного расчёта» при Play (глобальный playhead)
Раньше глобальный `playhead_ctrl` мог отправлять состояние в Python **несколько раз в секунду** (`send_hz=4`).

В Streamlit каждый такой апдейт означает **полный rerun всего приложения**, поэтому визуально это выглядело как «вечный пересчёт», даже если солвер не пересчитывался.

В R15:
- По умолчанию `send_hz=0` — **во время Play данные НЕ шлются в Python**, анимации идут **целиком в браузере** через `localStorage`.
- Добавлен UI‑переключатель **«Синхронизировать графики во время Play»**.
  - Если включить — можно задать частоту (1–10 Гц) и тогда маркеры на графиках будут двигаться, но это снова будет rerun.
  - Если выключено (по умолчанию) — графики обновляются при паузе/скраббинге.

### 1.2 Логирование: playhead + выбор бэкенда анимации
В `pneumo_solver_ui/logs/` теперь дополнительно пишется:
- `playhead_update` — изменение idx/t/playing (с троттлингом по изменениям состояния)
- `anim_backend_selected` — какой режим анимации выбран: `component` / `fallback`

Также добавлены **объединённые** логи:
- `logs/ui_combined.log`
- `logs/metrics_combined.jsonl`

При этом **пер‑сессионные** файлы (`ui_YYYY...pid...log`, `metrics_...jsonl`) сохраняются — они удобны для точечной диагностики.

### 1.3 Калибровка / Autopilot: пути стали понятнее и совместимее
Раньше часть UI писала в `workspace/calibration_runs`, а часть пайплайнов искала в `pneumo_solver_ui/calibration_runs`.

В R15:
- `calibration_runs/` теперь считается **канонической** папкой для oneclick/autopilot.
- `workspace/osc` остаётся папкой по умолчанию для осциллограмм, но в UI появился параметр **osc_dir_path**:
  - можно указать любой локальный путь;
  - калибровка/автопилот будут читать NPZ/CSV из него;
  - экспорт NPZ из симуляции теперь тоже старается писать в текущий `osc_dir`.

### 1.4 Исправления устойчивости UI
- `safe_dataframe` теперь поддерживает `hide_index` без падения.

## 2) Что это даёт по твоим вопросам

- **«Play уходит в бесконечный расчёт»** — теперь по умолчанию Play идёт без server rerun.
- **«Это глобальный playhead или fallback matplotlib?»** — теперь это видно по логам (`anim_backend_selected`, `playhead_update`).
- **«Почему не логируешь?»** — логирование расширено + есть объединённые файлы.

## 3) Важные файлы и папки

- `pneumo_solver_ui/components/playhead_ctrl/index.html` — playhead, теперь умеет `send_hz=0`.
- `pneumo_solver_ui/calibration_runs/` — результаты oneclick/autopilot.
- `pneumo_solver_ui/workspace/osc/` — осциллограммы NPZ/CSV по умолчанию (можно переопределить).
- `pneumo_solver_ui/logs/ui_combined.log` + `metrics_combined.jsonl` — объединённые логи.
