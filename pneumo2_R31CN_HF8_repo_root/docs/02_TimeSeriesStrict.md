# Time-series входы (road_csv / axay_csv) и режим strict

## Проблема
Ранее в worker (`opt_worker_v3_margins_energy.py`) ошибки чтения/компиляции временных рядов (CSV) подавлялись.
Это приводило к опасной ситуации: тест мог считаться по «дефолтным» функциям, хотя пользователь ожидал сценарий из CSV.

## Решение в v6.52
- Ошибка компиляции фиксируется в метриках (`ts_compile_ok`, `ts_compile_error`).
- По умолчанию включён строгий режим: `timeseries_strict=True`.
  В строгом режиме ошибка компиляции считается фатальной и тест падает (это безопаснее).

## Как отключить strict для конкретного теста
В записи теста (suite) добавьте поле:

```json
{"timeseries_strict": false}
```

Тогда при ошибке чтения CSV тест продолжит выполняться, но в метриках останется отметка о проблеме.

## Почему strict включён по умолчанию
- Ошибка во входных данных не должна превращаться в «молчаливое» изменение сценария.
- Для оптимизации это критично: иначе можно получить «лучшие параметры» для другого входа.
