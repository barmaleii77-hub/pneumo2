## Роль в базе знаний

V6 является отчётным аудитом GUI без кода и без готовых исправлений. Он не
заменяет V38 и V5: V38 остаётся текущим GUI/TZ/spec каноном, V5 остаётся
слоем проверки окон и доказательности. V6 уточняет пользовательский маршрут:
проект уже desktop-first по канону, но живой интерфейс всё ещё местами выглядит
переходным, перегруженным и недостаточно иерархичным.

## Главные выводы

1. Основная проблема не в нехватке кнопок, а в слабой иерархии пользовательского
   маршрута после запуска.
2. Самый понятно объяснённый путь: подробный расчёт -> актуальный файл анимации
   -> просмотр движения в аниматоре.
3. Самые слабые зоны: сценарии колец и единый путь диагностики.
4. `Редактор кольца` должен быть первоклассным верхнеуровневым входом для
   сценариев, а не только канонически описанным окном.
5. Диагностика должна иметь одно доминирующее действие `Собрать диагностику`;
   вспомогательные отправка, просмотр и файлы не должны конкурировать с ним на
   первом уровне.
6. Встроенное сравнение результатов и отдельное окно сравнения не должны
   выглядеть как два равнозначных первых маршрута.
7. Для графики, анимации, схемы и результатов нужны крупные видимые состояния
   достоверности: подтверждено расчётом, подтверждено исходными данными,
   приближённо, недоступно.
8. Каждая стартовая кнопка должна объяснять: что откроется, когда это нужно и
   что пользователь сможет сделать внутри.
9. Сложные инженерные и вычислительные настройки оптимизации должны быть
   скрыты до явного запроса пользователя на расширенные параметры.
10. Окна, которые в V6 отмечены только как точки запуска, нельзя считать
    визуально принятыми без отдельной проверки их внутреннего интерфейса.

## Ограничения доказательности

V6 не закрывает внутренний UX окон, если в таблицах они отмечены как
`launchpoint_only`. Это особенно важно для Desktop Mnemo, диагностики,
отправки результатов, ввода исходных данных, центра проверки и набора
desktop-инструментов.

`Редактор кольца` признан критичным пользовательским входом, но V6 не доказывает
его фактическую первоклассную визуальную позицию в текущем runtime.

## Отсутствие противоречий

V6 не противоречит V38 и V5. Он усиливает текущий курс:

- desktop GUI вместо WEB-first развития;
- один понятный пользовательский маршрут вместо россыпи равных кнопок;
- первоклассный `Редактор кольца`;
- единая диагностика через `Собрать диагностику`;
- разделение встроенного сравнения и отдельного окна сравнения;
- крупные состояния достоверности вместо служебных статусов;
- отказ от вывода внутренних технических слов в операторский интерфейс.
