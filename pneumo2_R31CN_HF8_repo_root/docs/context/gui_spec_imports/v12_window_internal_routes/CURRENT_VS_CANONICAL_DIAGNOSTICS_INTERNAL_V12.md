# Диагностика — внутренний action→feedback маршрут

## Current evidence
- окно подтверждено как launchpoint и subsystem, но не как полностью раскрытая current-window composition;
- в живом маршруте существует конкурирующий вход `GUI отправки результатов`.

## Current problem
Пользователь не понимает, что первично:
- собрать архив,
- или сразу отправлять результаты.

## Canonical route
1. Открыть `Диагностику` из дерева.
2. Увидеть текущий health summary.
3. Нажать `Собрать диагностику`.
4. Получить progress/status на этом же экране.
5. Получить готовый bundle: путь, свежесть, размер, хэш.
6. Только после этого получить вторичный выбор:
   - открыть папку,
   - скопировать путь,
   - отправить архив.

## Mandatory feedback
- `Сборка диагностики запущена`
- `Архив диагностики готов`
- `Собран fallback bundle` — если helper/runtime деградировал
- `Архив устарел` — если bundle не соответствует текущей сессии

## Canonical semantic rewrites
- `GUI диагностики` → `Диагностика`
- `GUI отправки результатов` → `Отправить архив диагностики`
- `latest_send_bundle.zip` → `Последний архив диагностики`
