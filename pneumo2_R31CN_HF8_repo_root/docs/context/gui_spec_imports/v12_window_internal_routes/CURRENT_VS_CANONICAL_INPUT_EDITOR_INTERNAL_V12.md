# Исходные данные проекта — внутренний action→feedback маршрут

## Current evidence
- окно подтверждено как точка входа, но не как живо раскрытая current-window composition.

## Current problem
Пользователь может редактировать числа, но не всегда получает ясную визуальную обратную связь о структуре подвески и логике активированных режимов.

## Canonical route
1. Открыть `Исходные данные проекта` из дерева.
2. Выбрать группу параметров.
3. Изменить значение.
4. Немедленно увидеть обновление графического двойника.
5. Увидеть карточки смыслового состояния:
   - две пружины на угол,
   - выравнивание активно,
   - на сколько выровнено,
   - симметрия включена/выключена.
6. Нажать `Проверить`.
7. Получить валидацию.
8. Нажать `Сохранить`.

## Mandatory feedback
- `Есть несохранённые изменения`
- `Проверка пройдена`
- `Есть предупреждения по геометрии или упаковке`
- `На угол установлены две пружины`
- `Остаточная разница после выравнивания: ... мм`

## Canonical semantic rewrites
- `Редактор исходных данных` → `Исходные данные проекта`
- `машинка` / неясная автоматика → `Автоматический подбор для статического положения`
- `лево/право` без контекста → `Левый борт / Правый борт`
