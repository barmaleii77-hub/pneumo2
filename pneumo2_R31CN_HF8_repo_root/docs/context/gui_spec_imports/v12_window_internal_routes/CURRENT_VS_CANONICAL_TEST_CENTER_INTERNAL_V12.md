# Набор испытаний — внутренний action→feedback маршрут

## Current evidence
- окно подтверждено как точка входа, но не как живо раскрытая current-window composition.

## Current problem
Пользователь может принять это окно за второй редактор сценариев, если ему не объяснить связь с редактором кольца.

## Canonical route
1. Открыть `Набор испытаний` из дерева.
2. Увидеть таблицу испытаний.
3. Понять по колонке и badge, какие строки связаны с редактором кольца.
4. Отредактировать stage/dt/t_end/targets только как test-context.
5. Проверить набор.
6. Сохранить validated snapshot.
7. При stale-link вернуться в `Редактор кольца`.

## Mandatory feedback
- `Набор испытаний проверен`
- `Контекст сценария устарел`
- `Отсутствует sidecar-файл`
- `Стадия первого входа недопустима`
- `Набор испытаний сохранён`

## Canonical semantic rewrites
- `Центр тестов` → `Набор испытаний`
- `stage` → `Стадия первого входа`
- `suite` → `Набор испытаний`
