# Подробное сравнение результатов — внутренний action→feedback маршрут

## Current evidence
- compare существует и как embedded compare, и как отдельный viewer;
- текущее окно подтверждено как launchpoint, но не как живо раскрытая внутренняя композиция.

## Current problem
Пользователь не понимает, когда нужно:
- просто быстрое встроенное сравнение,
- а когда — отдельное окно.

## Canonical route
1. Открыть `Подробное сравнение результатов` из дерева или из `Анализа`.
2. Выбрать режим сравнения.
3. Выбрать левый и правый объект.
4. Получить предупреждение о `current / historical / stale / contract mismatch`, если оно есть.
5. Нажать `Открыть сравнение`.
6. Переключать представления: overlay, таблица различий, метрики.

## Mandatory feedback
- `Выберите оба объекта сравнения`
- `Контракты сравнения различаются`
- `Сравнение построено`
- `Сравнение недоступно: отсутствует артефакт`

## Canonical semantic rewrites
- `Compare Viewer` → `Подробное сравнение результатов`
- `Открыть Compare Viewer` → `Открыть подробное сравнение`
- `compare` → `сравнение результатов`
