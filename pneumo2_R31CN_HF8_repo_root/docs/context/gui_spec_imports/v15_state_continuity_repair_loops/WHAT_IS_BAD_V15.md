# Что плохо в текущем состоянии относительно канона V15

1. Current UI слишком часто оставляет пользователя в неявном состоянии:
   непонятно, current это context или historical, свежий ли export/snapshot/bundle.
2. Есть риск, что stale и degraded будут ощущаться как “просто всё работает”, пока пользователь не ошибётся.
3. Без жёстких repair-loop правил:
   - diagnostics спорит с send-results,
   - compare спорит с embedded compare,
   - stale suite спорит с ring-source,
   - animator спорит с provenance.
4. Если не сохранить selection/context, пользователь после возврата оказывается на пустом экране и теряет нить.
5. Много важных состояний пока существуют в проектном каноне, но не всегда материализованы как видимый UI-marker.
