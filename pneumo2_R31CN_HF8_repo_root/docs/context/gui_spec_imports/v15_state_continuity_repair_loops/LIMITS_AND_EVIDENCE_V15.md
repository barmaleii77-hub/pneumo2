# Честные границы V15

Это **report-only canonical layer**.

Что здесь утверждается:
- как должны быть устроены state continuity и repair loops в целевом GUI;
- какие состояния обязаны быть user-visible;
- куда пользователь должен возвращаться для починки проблемы.

Что здесь не утверждается:
- что current runtime уже полностью реализует эти состояния;
- что все current внутренности окон живо раскрыты и проверены;
- что все markers уже подтверждены в реальном Windows runtime.

То есть это **канон поведения и маршрутов восстановления**, а не утверждение о полном текущем соответствии.
