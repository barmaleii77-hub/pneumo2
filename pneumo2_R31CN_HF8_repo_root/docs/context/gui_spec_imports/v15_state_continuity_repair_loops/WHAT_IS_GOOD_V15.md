# Что хорошо в каноне V15

1. Проблема описывается не как “есть кнопка / нет кнопки”, а как **жизненный цикл состояния пользователя**.
2. Upstream/downstream связи становятся явными:
   `Исходные данные -> Редактор кольца -> Набор испытаний -> Baseline -> Optimization -> Analysis -> Animator -> Diagnostics`.
3. Repair-loop не ломает основную цепочку, а возвращает пользователя **в правильную точку починки**.
4. Критические user-visible состояния больше не прячутся:
   - две пружины на угол,
   - выравнивание пружинами,
   - остаточная разница,
   - historical/current,
   - degraded truth.
5. Канон совместим с прямым открытием из дерева и dock-first shell.
