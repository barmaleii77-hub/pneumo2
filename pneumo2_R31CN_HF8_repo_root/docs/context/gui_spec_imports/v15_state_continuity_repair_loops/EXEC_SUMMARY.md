# Краткий итог V15

Это **report-only слой**, без кода и без правок проекта.

Что сделано:
- следующий слой после V14: описано не только **как окно открыть из дерева и где оно dock / top-level**, но и
  **как оно должно сохранять состояние**, **как вести пользователя при stale/dirty/mismatch**, и
  **как возвращать его в правильный узел pipeline без потери контекста**;
- зафиксированы repair-loops для ключевых состояний:
  - `dirty`
  - `invalid`
  - `stale link`
  - `context stale`
  - `baseline/source mismatch`
  - `objective-contract mismatch`
  - `truth degraded`
  - `bundle stale`;
- отдельно описан слой обязательной когнитивной видимости:
  пользователь должен **не просто изменить данные**, а **увидеть глазами, что именно изменилось**.

Главная мысль:
проблема уже не только в том, что окно надо открыть одним действием.
Проблема в том, что после открытия пользователь должен:
1. увидеть **какой контекст сейчас открыт**;
2. понять **можно ли доверять текущему состоянию**;
3. знать **что именно сломано или устарело**;
4. иметь **один основной repair-route**;
5. вернуться назад **в то же место pipeline**, а не на голый стартовый экран.
