Самое важное из моего live прогона:

### A. Acceptance drift в cylinder/animator слое
Есть реальное противоречие внутри текущего дерева:
- `tests/test_r44_cylinder_packaging_contract_and_animator.py` падает на ожидании, что rod должен начинаться уже **снаружи** housing.
- Но текущая живая реализация в `desktop_animator/geom3d_helpers.py` делает другое: при текущем честном fallback рисует
  - `housing_seg = top -> gland/body shell or full shell`
  - `rod_seg = piston_center -> bot`
  и отдельно держит прозрачную оболочку + внутренний rod/piston layer.
- Это согласуется с более поздними тестами `r46` и частью `r52/r57`, но не согласуется со старым `r44`.

То есть сейчас в live tree есть **нечистый acceptance слой**, и он прямо на P0-фронте cylinder packaging.

### B. Animator regression slice не зелёный целиком
Ещё 3 фейла идут из `r52/r57`:
- это source-string / signature expectations против текущего `desktop_animator/app.py`;
- по характеру похоже на drift тестов относительно текущего файла, а не на уже доказанную физическую регрессию;
- но честный статус от этого не меняется: **green acceptance тут сейчас нет**.

### C. Не подтверждён end-to-end Windows слой
Я здесь **не** подтвердил:
- свежий Windows visual acceptance,
- свежий SEND bundle именно после R31CN/следующего патча,
- measured browser perf trace acceptance,
