# Краткий итог V14

Это **report-only слой**, без кода и без правок проекта.

Что сделано:
- следующий слой после V13: окна описаны не только как внутренние маршруты, но и как
  **точки входа из левого дерева**, **dock / top-level элементы**, **контекстно-зависимые поверхности**
  и **узлы возврата без потери состояния**;
- для четырёх проблемных окон и связанных рабочих пространств зафиксировано:
  - как именно окно открывается из дерева;
  - должно ли оно открываться в центре, как отдельное top-level окно или как dock-поверхность;
  - какой контекст обязано наследовать;
  - какие маркеры уверенности пользователь должен увидеть в первые секунды;
  - как пользователь возвращается назад без повторного выбора всего маршрута.

Главная мысль:
проблема уже не только в том, что окно "есть".
Проблема в том, что пользователь должен:
1. открыть окно **одним действием из дерева**;
2. увидеть **главный смысл окна сразу**;
3. не потерять selection/context;
4. вернуться назад **в то же состояние pipeline**, а не на голый стартовый экран.
