# Что хорошо в текущем направлении

1. Канон проекта уже требует видимую основную цепочку, first-class диагностику и единый сценарный source-of-truth.
2. Логика прямого открытия из дерева хорошо ложится на desktop-first shell.
3. Проблемные окна уже удалось перевести из статуса «есть кнопка» в операционные поверхности с первым экраном и главным действием.
4. Compare и Diagnostics уже достаточно зрелые по смыслу, чтобы не изобретать их заново, а лишь правильно встроить в иерархию.
