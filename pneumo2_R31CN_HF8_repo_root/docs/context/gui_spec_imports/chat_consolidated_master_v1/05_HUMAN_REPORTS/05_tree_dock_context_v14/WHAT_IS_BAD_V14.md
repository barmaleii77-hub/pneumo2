# Что плохо в текущем состоянии

1. Пользователь всё ещё может столкнуться с плоским launcher-shell, где окна выглядят как почти равноправные кнопки.
2. Открытие некоторых окон сейчас мыслится как отдельный launchpoint, а не как продолжение основного маршрута.
3. Контекст выбора может рваться: пользователь открывает сравнение, диагностику или набор испытаний и вынужден заново восстанавливать, откуда он пришёл.
4. Для пользователя мало явно зафиксировано, что является:
   - top-level workspace,
   - advanced secondary surface,
   - nested action внутри другого окна.
5. Роль «Отправка результатов» всё ещё слишком легко начинает спорить с ролью «Диагностика».
