# EXEC SUMMARY — V11

Это точечный current-vs-canonical pass по окнам, которые в текущем live-маршруте
подтверждены в основном как launchpoint-only:

- GUI диагностики
- Compare Viewer
- Редактор исходных данных
- Центр тестов

Цель шага:
1. честно отделить то, что подтверждено в current;
2. не притворяться, что внутренности окон уже открыты;
3. зафиксировать каноническую роль каждого окна;
4. убрать графовые конфликты маршрутов;
5. выдать семантические переписывания названий, кнопок и сообщений.

Главный вывод:
- в current эти окна ещё не образуют понятный маршрут;
- в canonical их роли ясны, но их надо развести по иерархии shell;
- самое опасное сейчас — ложная равноправность маршрутов:
  - GUI диагностики ↔ GUI отправки результатов
  - embedded compare ↔ Compare Viewer
  - Редактор исходных данных ↔ тяжелая домашняя поверхность
  - Центр тестов ↔ сценарный центр продукта

Итоговая рекомендация:
- Диагностика, Исходные данные проекта и Набор испытаний должны быть first-class entrypoints;
- Compare Viewer должен остаться прямым окном из дерева, но как advanced-ветка Анализа, а не как конкурирующий основной compare-route;
- все четыре окна должны открываться напрямую из левого дерева и через поиск команд, без промежуточного “центра окон”.
