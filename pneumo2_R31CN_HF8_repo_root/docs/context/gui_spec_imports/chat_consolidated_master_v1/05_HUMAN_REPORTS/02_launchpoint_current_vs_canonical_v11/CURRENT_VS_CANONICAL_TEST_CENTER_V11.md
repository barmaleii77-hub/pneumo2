# Центр тестов — current vs canonical pass

## Current, что честно подтверждено
- Есть отдельная точка входа `Центр тестов`.
- Current internal UX не раскрыт; статус: `launchpoint_only`.
- На launcher-уровне окно выглядит как ещё один параллельный сценарный центр.

## Что пользователь понимает сейчас плохо
- Это редактор сценариев или редактор набора испытаний.
- Можно ли здесь менять геометрию дороги.
- Как это связано с редактором кольца.
- Какие поля здесь главные: stage, priority, enabled, dt, t_end, sidecars, validation.

## Canonical role
- First-class top-level workspace.
- Каноническое имя: `Набор испытаний`.
- Это consumer сценариев, а не их источник.
- Геометрия сценария приходит только из `Редактора кольца`.

## Каким должен быть первый экран
1. Центральная таблица набора испытаний:
   - enabled
   - name
   - type
   - stage
   - priority
   - dt
   - t_end
2. Левый browser:
   - наборы / группы / история snapshot
3. Правый inspector:
   - validation
   - sidecar artifacts
   - provenance
4. Явная ссылка-направление:
   - `Редактировать сценарий в редакторе кольца`

## Что исправлять в семантике
- `Центр тестов` → `Набор испытаний`
- `stage` → `Стадия первого входа`
- `priority` → `Приоритет`
- `sidecar` → `Связанные файлы сценария`
- `validation` → `Проверка набора испытаний`

## Dock/window contract
- direct-open node в дереве;
- центральная таблица + правый validation inspector;
- нижняя полоса состояния с ошибками набора;
- запрет превращать окно в второй редактор сценарной геометрии.
