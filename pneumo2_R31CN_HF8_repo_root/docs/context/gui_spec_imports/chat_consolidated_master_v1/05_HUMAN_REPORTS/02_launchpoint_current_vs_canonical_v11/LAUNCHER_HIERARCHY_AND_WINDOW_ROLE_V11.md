# Launcher-shell и роли окон — V11

## Top-level first-class окна / workspace
- Исходные данные проекта
- Редактор кольца / сценариев
- Набор испытаний
- Базовый прогон
- Оптимизация
- Анализ результатов
- Аниматор
- Диагностика

## Direct-open advanced окна / routes
- Подробное сравнение результатов
- Мнемосхема
- Инструменты

## Что нельзя оставлять как равноправные стартовые пути
- `GUI отправки результатов`
- `Compare Viewer` как основной compare-route
- `Центр desktop-инструментов` как обязательный шаг
- `Desktop Mnemo` как ранний обязательный путь

## Принцип
Окно должно открываться:
- напрямую из левого дерева;
- напрямую из поиска команд;
- без промежуточного окна-хаба.

## Для четырёх анализируемых окон
- `Диагностика` — first-class top-level
- `Подробное сравнение результатов` — advanced direct-open из ветки Анализа
- `Исходные данные проекта` — first-class top-level
- `Набор испытаний` — first-class top-level
