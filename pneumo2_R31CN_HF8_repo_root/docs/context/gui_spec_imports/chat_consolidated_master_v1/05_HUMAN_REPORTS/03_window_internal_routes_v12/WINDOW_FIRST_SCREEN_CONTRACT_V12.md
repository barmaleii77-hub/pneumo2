# First-screen contracts — V12

## 1. Диагностика

### Цель первого экрана
Понять за 3–5 секунд:
- здорова ли текущая сессия;
- есть ли уже свежий архив диагностики;
- что попадёт в новый архив;
- какой следующий шаг: **собрать**, **открыть**, **скопировать путь**, **отправить**.

### Первый экран должен содержать
- заголовок: **Диагностика**
- один primary action: **Собрать диагностику**
- карточку состояния системы
- карточку последнего архива: путь, свежесть, размер, время
- карточку состава bundle
- карточку self-check
- вторичную команду: **Отправить архив диагностики** — только после готового bundle

## 2. Подробное сравнение результатов

### Цель первого экрана
Понять за 3–5 секунд:
- что именно сейчас сравнивается;
- `current / historical / stale` контекст;
- достаточно ли быстрого встроенного compare или уже открыт advanced-viewer.

### Первый экран должен содержать
- заголовок: **Подробное сравнение результатов**
- selector режима сравнения:
  - baseline против выбранного прогона
  - прогон против прогона
  - артефакт против артефакта
- два явных слота выбора левой и правой стороны
- badge контекста: `текущий`, `исторический`, `несовпадение контракта`
- primary action: **Открыть сравнение**

## 3. Исходные данные проекта

### Цель первого экрана
Понять за 3–5 секунд:
- что это master-copy входов проекта;
- в каком состоянии редактирование: черновик, сохранено, проверено, есть предупреждения;
- где на схеме отражается текущее изменение.

### Первый экран должен содержать
- заголовок: **Исходные данные проекта**
- левое дерево групп параметров
- центральную форму выбранной группы
- синхронный графический двойник
- карточки когнитивной видимости:
  - **На угол установлены две пружины**
  - **Выравнивание пружинами активно**
  - **Остаточная разница после выравнивания, миллиметры**
  - **Зеркальная симметрия включена / отключена**
- primary actions: **Сохранить** и **Проверить**

## 4. Набор испытаний

### Цель первого экрана
Понять за 3–5 секунд:
- какие испытания реально входят в расчёт;
- на какой стадии они впервые вступают;
- что взято из редактора кольца, а что — локальный тестовый контекст.

### Первый экран должен содержать
- заголовок: **Набор испытаний**
- таблицу строк испытаний
- явные колонки:
  - включено
  - имя
  - тип
  - стадия первого входа
  - шаг интегрирования, секунды
  - время моделирования, секунды
  - sidecar/сценарий
- badge `связано с редактором кольца`
- warning `контекст сценария устарел`
- primary actions: **Проверить набор** и **Сохранить снимок набора**
