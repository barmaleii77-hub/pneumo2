# Первые 15 минут пользователя — целевая логика

## Как это должно читаться
Пользователь запускает приложение и сразу видит:
1. **Исходные данные** — где задать или проверить master-copy входов.
2. **Сценарии / Редактор кольца** — где задать сценарный источник истины.
3. **Набор испытаний** — где выбрать, что реально пойдёт в расчёт.
4. **Базовый прогон** — где быстро получить baseline.
5. **Оптимизация** — где есть один рекомендуемый режим и один advanced режим.
6. **Анализ** — где есть быстрое сравнение.
7. **Анимация** — отдельное окно просмотра результата.
8. **Диагностика** — единая заметная команда и отдельная рабочая поверхность.

## Что не должно спорить с этим маршрутом
- Compare Viewer не должен спорить с встроенным compare.
- GUI отправки результатов не должен спорить с Diagnostics.
- Desktop Mnemo не должен спорить с Animator.
- Центр desktop-инструментов не должен выглядеть как первый обязательный шаг.
