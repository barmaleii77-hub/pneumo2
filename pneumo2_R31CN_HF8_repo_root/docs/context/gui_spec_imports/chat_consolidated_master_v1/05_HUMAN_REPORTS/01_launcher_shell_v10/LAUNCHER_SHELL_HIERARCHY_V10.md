# Иерархия launcher-shell — V10

## 1. Что должно быть top-level
Top-level должны остаться только те окна и рабочие поверхности, которые образуют основную инженерную цепочку:
- Главный shell проекта
- Редактор исходных данных
- Редактор кольца / сценариев
- Центр тестов
- Базовый прогон
- Оптимизация
- Анализ результатов
- Desktop Animator
- GUI диагностики

## 2. Что не должно быть равно этим пунктам по приоритету
Следующие вещи полезны, но не должны быть первыми равноправными кнопками:
- GUI отправки результатов
- Compare Viewer
- Desktop Mnemo
- Центр desktop-инструментов

## 3. Почему
### GUI отправки результатов
Это не самостоятельный стартовый путь.  
Это продолжение уже собранной диагностики.

### Compare Viewer
Это не основной compare-route.  
Основной compare должен жить внутри Анализа, а Viewer — быть расширенным режимом.

### Desktop Mnemo
Окно полезно как дополнительная инженерная поверхность, но не как один из первых пользовательских шагов.

### Центр desktop-инструментов
Это service-hub, а не обязательный шаг основного сценария.

## 4. Как должен читать shell обычный пользователь
Стартовая поверхность должна отвечать на один вопрос:
**«Что делать сначала?»**

Ответ должен быть виден прямо в shell:
1. Исходные данные
2. Сценарии
3. Набор испытаний
4. Базовый прогон
5. Оптимизация
6. Анализ
7. Анимация
8. Диагностика
