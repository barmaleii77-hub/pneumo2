# Как поправить — V10

1. Оставить на старте один доминирующий маршрут из 8 шагов: Исходные данные → Сценарии → Набор испытаний → Базовый прогон → Оптимизация → Анализ → Анимация → Диагностика.
2. Поднять Редактор кольца в явный top-level сценарный центр и прямо пометить его как единственный источник сценариев.
3. Оставить одну заметную команду «Собрать диагностику», а отправку результатов увести внутрь Diagnostics как следующий шаг.
4. Сделать встроенное сравнение основным compare-режимом, а Compare Viewer — расширенным режимом из Анализа.
5. Обозначить Редактор исходных данных как master-copy входов, а Центр тестов — как consumer сценариев.
6. Desktop Mnemo и Центр desktop-инструментов понизить до advanced-уровня и объяснять их роль короткой фразой прямо в shell.
7. На стартовом экране добавить короткий текстовый маршрут 'Что делать сначала / потом / позже', а не только набор кнопок.
