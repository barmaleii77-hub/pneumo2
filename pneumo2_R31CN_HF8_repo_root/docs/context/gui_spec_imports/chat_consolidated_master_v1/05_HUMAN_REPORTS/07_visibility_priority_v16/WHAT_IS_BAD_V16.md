# Что плохо в current относительно V16

- слишком много значимых состояний остаются только в глубине полей, help или исторических знаний пользователя;
- shell может сообщать, где открыть окно, но недостаточно хорошо сообщает, **что именно пользователь обязан сразу понять**;
- в Inputs и Ring часть критичных смыслов можно легко пропустить, если смотреть только на числа;
- в Optimization часть неудач выполнения воспринимается как “зависло”, если underfill/gate/block не вытащены на главный экран;
- в Animator без крупного truth-marker красивая картинка продолжает визуально перевешивать honest-mode предупреждение;
- в Diagnostics всё ещё высок риск конкуренции между “собрать” и “отправить”, если не делать сильную primary/secondary иерархию.
