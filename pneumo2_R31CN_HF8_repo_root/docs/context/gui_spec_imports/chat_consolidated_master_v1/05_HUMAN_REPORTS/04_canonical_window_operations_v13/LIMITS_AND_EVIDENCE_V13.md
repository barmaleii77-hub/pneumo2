# Честные границы и доказанность

## Что проверено повторно
Для этого слоя перепроверены и использованы:
- report-only V10 launcher hierarchy;
- report-only V11 current-vs-canonical roles;
- report-only V12 internal routes;
- graph V19/V20 optimized subgraphs для `WS-INPUTS`, `WS-DIAGNOSTICS`, `WS-SUITE`, `WS-ANALYSIS`;
- исходные project prompt-файлы и connector-reconciled prompt.

## Что остаётся не доказанным в current runtime
Эти четыре окна по current runtime по-прежнему нельзя выдавать за полноценно открытые живые внутренности:
- Диагностика;
- Подробное сравнение результатов;
- Исходные данные проекта;
- Набор испытаний.

Для них этот пакет описывает **canonical internal route**, а не доказанный current-screen capture.

## Почему это всё равно полезно
Потому что уже сейчас можно:
- снять графовые конфликты ролей;
- снять конфликты названий;
- задать правильный first-screen contract;
- зафиксировать обязательные handoff и repair loops;
- снизить когнитивную стоимость переходов ещё до появления новых runtime-скриншотов.
