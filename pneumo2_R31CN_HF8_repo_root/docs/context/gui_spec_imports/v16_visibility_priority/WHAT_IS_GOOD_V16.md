# Что хорошо в V16

- впервые выделен не только маршрут, но и **иерархия видимости смыслов**;
- shell, inspector и message bar разведены по semantic roles;
- состояния вроде `две пружины на угол` и `выравнивание пружинами` больше не считаются “деталями для help”, а подняты в must-see слой;
- stale/current/historical/truth конфликты рассматриваются как отдельная визуальная политика, а не частные баги экранов;
- граф `VISIBILITY_ESCALATION_GRAPH_V16.dot` описывает эскалацию состояния от скрытого эффекта к явному repair-route.
