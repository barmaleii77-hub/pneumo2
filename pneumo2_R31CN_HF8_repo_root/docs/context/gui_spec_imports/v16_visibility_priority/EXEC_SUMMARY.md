# Краткий итог V16

Это **report-only слой**, без кода и без правок проекта.

Что сделано:
- следующий слой после V15: описано, **какие состояния должны быть видимы всегда**, какие только при конфликте, а какие можно уводить в inspector/help;
- зафиксирована **must-see политика** по основным workspace;
- добавлен shell-уровень для:
  - левого дерева как прямого launcher;
  - правого inspector/help;
  - нижнего status/progress strip;
  - message bar как места для stale/current/historical/truth конфликтов;
- отдельно выделены состояния, которые пользователь обязан понимать **в первые 3–5 секунд** после входа в workspace.

Главная мысль:
инженерный GUI должен показывать не всё подряд, а **правильное первое иерархическое приближение смысла**:
1. где я нахожусь;
2. что сейчас главное;
3. можно ли доверять текущему контексту;
4. что именно сломано, если есть конфликт;
5. куда идти чинить проблему без потери контекста.
