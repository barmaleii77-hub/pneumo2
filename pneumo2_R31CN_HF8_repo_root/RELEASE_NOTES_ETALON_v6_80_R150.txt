ETALON v6.80 — R150

Дата: 2026-02-19

Ключевые изменения (совместимость и контракт данных):

1) ABSOLUTE LAW: единый реестр параметров
- Добавлены документы:
  - 00_READ_FIRST__ABSOLUTE_LAW.md
  - 01_PARAMETER_REGISTRY.md
  - pneumo_solver_ui/contracts/param_registry.yaml
- Обновлён AI_INTEGRATION_PLAYBOOK.yaml: ИИ обязан начинать работу с чтения правил и реестра.

2) Запрет алиасов/дублей: канонический ключ скорости
- Введён единый канонический ключ скорости: vx0_м_с.
- Удалены/заменены места, где использовались разные имена одного и того же параметра (road_speed_mps / speed_mps / v0_м_с / скорость_м_с).
- В suite по умолчанию (default_suite.json) скорость хранится только как vx0_м_с.

3) NPZ/meta контракт: миграция только с предупреждением
- meta нормализуется в data_contract.py:
  - legacy‑ключи мигрируются в канонические с warning
  - legacy‑ключи удаляются (никаких дублей)

4) Экспорт результатов: NaN не подменяем на 0
- npz_bundle.py: nan_to_num удалён.
- NaN сохраняется как NaN, +/-inf заменяется на NaN.

5) Desktop Animator: сервисные производные значения + предупреждения
- s_world вычисляется строго как SERVICE/DERIVED из скорости (скорость_vx_м_с).
- При отсутствии сигнала скорости выдаётся warning и используется meta['vx0_м_с'] только как деградированный сервисный fallback.
- Загрузка road sidecar теперь логирует ошибки.

6) Логи обязательны
- Подключён diag/bootstrap.py в app.py и desktop_animator/main.py.
- Все подозрительные ситуации должны логироваться и объясняться, без «тихих» падений.
