# RELEASE_NOTES — ETALON_v6_80_R148

Дата: 2026-02-19

## Главное
- **Desktop Animator: убраны любые визуальные Z-базы/смещения (`vis_*`)** — отрисовка теперь использует **только** данные, пришедшие из расчетной модели (NPZ), без «поднятия» кузова/колес ради красоты.
- **2D/3D анимация синхронизирована с расчетом**: исправлена логика координат (колеса больше не вычисляются как `body + wheel`, если в NPZ уже даны абсолютные/совместимые координаты).

## Что исправлено
- Desktop Animator:
  - Удалены все «визуальные» оффсеты/базы по Z (ранее добавлялись в DataBundle и применялись в Front/Side/3D).
  - FrontViewWidget/SideViewWidget: координаты колес и дороги берутся напрямую из колонок модели (`перемещение_колеса_*_м`, `дорога_*_м`, `рама_*_z_м`).
  - GLViewWidget (3D): признак «колесо в воздухе» теперь читается из колонок `колесо_в_воздухе_*` (а не из несуществующих `wheel_air_*`).

## Важно
- Если расчетная модель выдаёт координаты в относительном базисе (например, `*_rel0`), Animator отображает **ровно этот базис**. Для «физически абсолютной» геометрии (клиренс, абсолютная высота кузова над землей и т.д.) требуется экспортировать такие абсолютные координаты из модели явно.
