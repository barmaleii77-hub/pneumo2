# Desktop Animator — справка

> Desktop Animator — инженерное приложение визуализации результатов симуляции подвески/пневматики.
> Это **не CAD** и **не редактор**. Геометрия строится автоматически из параметров базы (колея/база/радиус колеса)
> и выходных сигналов симуляции.

## 1) Быстрый старт
1. **Откройте NPZ**: кнопка **Open NPZ…** или перетащите файл `.npz` прямо в окно приложения.
2. Нажмите **Play** или двигайте **ползунок кадра**.
3. Смотрите одновременно:
   - **3D** (рама + колёса + дорога),
   - **перед/зад** (крен),
   - **лево/право** (тангаж),
   - **Road HUD** (повороты/радиус/скорость),
   - **Telemetry** (пневматика/клапаны/расход/контакт/теплокарты).

## 2) Масштаб и нулевой уровень
- Во всех видах дорога рисуется в **реальных метрах** (один масштаб для всех 2D‑видов).
- **Нулевой уровень**: горизонтальная ровная дорога (z=0).
- На 2D‑видах показываются:
  - линия **нулевого уровня**,
  - уровень дороги **под каждым колесом**,
  - положение колеса и рамы (и их векторы v/a при включении).

## 3) Сплиты/панели (ресайз)
Панели разделены **перетаскиваемыми разделителями** (splitters).
- Тяните границы, чтобы дать больше места нужному виду (например, 3D или Telemetry).
- Положение разделителей сохраняется между запусками.

## 4) REL0 и ABS (важно!)
В логе часто есть два варианта сигналов:
- **ABS** — абсолютные значения (нужны для геометрии и корректной визуализации уровня дороги).
- **REL0** — значения относительно первого кадра (удобны для графиков/дельт).

**Закон проекта (ABSOLUTE-ONLY ANIMATION):** Desktop Animator всегда использует **ABS**
для отрисовки геометрии. Никаких скрытых baseline/offset/«нулевой позы» со стороны аниматора.

`*_rel0` допускаются для графиков/анализа и могут использоваться только явно
(например, в числовых оверлеях/таблицах), но не подставляются автоматически.

## 5) Подсказки и режим «Что это?»
- Справа есть блок **Подсказка** — он показывает объяснение для элемента под курсором.
- Кнопка **? (What’s This)** включает режим «покажи описание»: кликните по элементу интерфейса и получите расширенную справку.

## 6) Follow pointer (минимум ручных действий)
Если вы используете пайплайн, который пишет JSON‑файл `anim_latest.json`:
1. Укажите путь через кнопку **Pointer…** или перетащите `.json` в окно.
2. Включите **Follow** — Animator будет автоматически подхватывать последний NPZ.

## 7) Screenshot / экспорт кадров
Кнопка **Screenshot** сохраняет PNG текущего окна в:
`pneumo_solver_ui/workspace/exports/screenshots/`.

## 8) Автопроверки (Self-checks)
После загрузки NPZ запускаются быстрые проверки:
- монотонность времени,
- наличие ключевых каналов,
- sanity для ABS/REL0,
- грубая проверка wheel_z − road_z по радиусу.

Результат показывается в панели Telemetry и сохраняется рядом с NPZ как `*.selfcheck.json`.

---
Если что-то выглядит «странно», первым делом:
- проверьте REL0/ABS,
- посмотрите Self-checks,
- убедитесь, что в NPZ есть каналы дороги/колёс/рамы.
