# CPU Utilization (многопроцессный расчёт)

Коротко
-------
- Для CPU-bound расчётов Python должен использовать **процессы**, а не потоки (из-за GIL).
- При `jobs>1` важно избегать «оверсабскрипции» BLAS/OpenMP: каждый процесс не должен поднимать
  ещё один пул потоков на все ядра.

Практика для этого проекта
--------------------------
1) Оптимизатор (`opt_worker_v3_margins_energy.py`) использует `ProcessPoolExecutor`.
2) Если `jobs > flush_every`, то раньше пул мог простаивать.
   Теперь размер batch автоматически увеличивается до `max(flush_every, jobs)`,
   но **CSV всё равно пишется каждые `flush_every` строк**.
3) В дочерних процессах принудительно выставляется `OMP/MKL/OPENBLAS/..._NUM_THREADS=1`
   + (если установлен) `threadpoolctl.threadpool_limits(1)`.

Где настраивается
-----------------
- UI: «Параллельность (jobs)».
- Worker CLI: `--jobs`, `--flush_every`.
