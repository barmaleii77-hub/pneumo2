# Нулевая поза (нулевая позиция для графиков (и только как явный, опциональный слой для анимации))

**Нулевая поза** в проекте определяется как:

- дорога: ровная горизонтальная поверхность, `road = 0` в момент `t=0`;
- подвеска: статическое состояние (скорости и ускорения близки к нулю);
- штоки пневмоцилиндров: **в районе середины хода** (цель — 0.5 от полного хода) при заданных начальных параметрах пневматики;
- все относительные перемещения для графиков/анимации строятся **от этой нулевой позы**.

## Что делает приложение

1. **Static trim (статическая подстройка)**
   - Если включено `static_trim_enabled=true`, модель выполняет подстройку начальных координат (heave/roll/pitch + углы/смещения колёс) так, чтобы:
     - ускорения в начале были близки к нулю;
     - геометрические связи не были нарушены;
     - ход подвески относительно рамы (`dw`) был близок к нулю (т.е. к «номиналу»).

2. **Rel0-представление для графиков (и только как явный, опциональный слой для анимации)**
   - Для ключевых колонок создаются и используются `*_rel0`, где из значения вычтено значение в `t=0`.
   - Это гарантирует: на графиках и в анимации «нулевая поза» становится нулём.

3. **Автосамопроверка нулевой позы (postflight)**
   - В отчёт `autoself_post_json` добавлен пункт `zero_pose`.
   - В нём фиксируются значения в `t=0`:
     - `дорога_*_м`;
     - `рама_угол_*_z_м`;
     - `перемещение_колеса_*_м`;
     - `сжатие_подвески_колесо_*_м`;
     - `положение_штока_*_м` и доля хода `rod_*_frac`.

## Настраиваемые пороги

Параметры берутся из `base.json` (если ключ не задан — используются безопасные значения по умолчанию):

- `zero_pose_target_stroke_frac` (по умолчанию `0.5`) — целевая доля хода штока в `t=0`.
- `zero_pose_tol_stroke_frac` (по умолчанию `0.2`) — допуск по доле хода.
- `zero_pose_road_tol_m` (по умолчанию `1e-6`) — допуск для `дорога_*_м` в `t=0`.

## Интерпретация результата

- `severity = info` — всё в норме.
- `severity = warn` — нулевая поза допустима, но есть отклонения (например, шток заметно не в середине хода).
- `severity = error` — нарушение базовых ограничений (например, доля хода штока вне `[0..1]`, NaN).

## Где это смотреть в UI

После прогона теста:

- блок **«Самопроверки (детальный прогон)»**
- строка **«Нулевая поза: OK / ПРОБЛЕМА»**
- подробности в экспандере **«Детали самопроверок»** → **«Нулевая поза (t=0)»**.


**ВАЖНО (ABSOLUTE LAW):**

* Desktop Animator и web‑анимация **никогда не подставляют автоматически** `*_rel0` вместо абсолютных сигналов.
* Если нужны смещения/rel0 — это делается **явно** и помечается как `SERVICE/DERIVED`.
