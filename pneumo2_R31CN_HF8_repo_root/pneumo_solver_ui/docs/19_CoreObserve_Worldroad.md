# 19) Core Observe (WorldRoad v9/v10)

## Зачем это нужно

В проекте есть два противоречащих требования:

1) **Надёжная верификация и UI** — удобно иметь `df_main` со всеми колонками и Excel‑выгрузки.
2) **Оптимизация/калибровка** — хочется дешёвую оценку KPI без построения больших DataFrame.

Режим `compile_only=True` (см. `18_CoreRHS_CompileOnly.md`) отдаёт ядро ОДУ (`rhs`, `rk2_step`, ...).
В релизе **v6.44** добавлена функция **`observe(state, t)`**, которая вычисляет ключевые наблюдаемые
величины в точке времени *без* построения `df_main`.

Это позволяет:

- считать KPI **онлайн** (streaming) во время интегрирования;
- легко проверять корректность трактовки перемещений (P0):
  * колесо относительно дороги,
  * колесо относительно рамы,
  * шток относительно рамы/колеса;
- подготовить код к будущему **полностью дифференцируемому** пайплайну (JAX/Diffrax/CasADi и т.п.).

## Где реализовано

Файл:
- `pneumo_solver_ui/model_pneumo_v9_mech_doublewishbone_worldroad.py`

Shim‑модель v10 (`model_pneumo_v10_mech_doublewishbone_worldroad_iso6358.py`) импортирует v9,
поэтому изменения применяются автоматически и к «v10».

## API

Получить ядро:

```python
ctx = simulate(params, test, dt=..., t_end=..., compile_only=True)
state = ctx['state0']
step = ctx['rk2_step']
observe = ctx['observe']
```

Наблюдение:

```python
obs = observe(state, t)
```

Аргументы:
- `include_p=False` — добавить в `obs` полный вектор давлений `p_Па` и объёмы `V_м3`.
- `include_m=False` — добавить в `obs` полный вектор масс `m_кг`.

### Что возвращает `observe`

Возвращается словарь.

1) **Векторные поля** (numpy‑массивы длины 4):
- `zw_center`, `zw_contact` — координаты центра колеса и точки контакта
- `z_road`, `z_road_dot` — профиль дороги и скорость профиля
- `penetration`, `penetration_dot` — пенетрация по точке контакта и её скорость
- `F_tire` — нормальная сила шины
- `wheel_in_air` — флаг отрыва (int) по условию `F_tire <= 1 Н`
- `z_body`, `z_body_dot` — высота рамы в точках колёс
- `s_C1`, `s_C2`, `s_dot_C1`, `s_dot_C2` — положения/скорости штоков цилиндров

2) **Поля совместимые с df_main** (по углам `ЛП/ПП/ЛЗ/ПЗ`):
- `перемещение_колеса_ЛП_м`, `контакт_колеса_ЛП_м`, `дорога_ЛП_м`, ...
- `пенетрация_шины_ЛП_м`, `нормальная_сила_шины_ЛП_Н`, `колесо_в_воздухе_ЛП`, ...
- `положение_штока_Ц1_ЛП_м`, `положение_штока_Ц2_ЛП_м`, ...

3) **Давления в ключевых узлах**:
- `давление_ресивер1_Па`, `давление_ресивер2_Па`, `давление_ресивер3_Па`, `давление_аккумулятор_Па`

4) **Ускорения (из RHS)**

`observe` отдаёт ускорения напрямую из RHS:
- `ускорение_рамы_z_м_с2_rhs`
- `ускорение_крен_phi_рад_с2_rhs`
- `ускорение_тангаж_theta_рад_с2_rhs`
- `ускорение_колеса_м_с2_rhs` (массив 4)

⚠️ В `df_main` ускорения вычисляются через `np.gradient` от скоростей — это может чуть отличаться
(особенно на грубой сетке), поэтому для сравнения с `df_main` лучше использовать колонки
перемещений/скоростей/сил, а для KPI — RHS‑ускорения.

## Тесты

Добавлен тест:
- `tests/test_worldroad_core_observe_equivalence.py`

Он проверяет, что `observe()` воспроизводит ключевые колонки `df_main` на нескольких шагах
для ненулевого профиля дороги.
