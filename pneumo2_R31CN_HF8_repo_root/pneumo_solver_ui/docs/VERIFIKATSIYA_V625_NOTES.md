# Verifikatsiya v6.25 — Notes (energy/entropy verification)

Дата: 2026-01-28 (Europe/Amsterdam)

## Что было найдено

### 1) Дрейф внутренней энергии газа в термомодели (u-модель)
В `model_pneumo_v8_energy_audit_vacuum.py` (режим `термодинамика != isothermal`) состояние содержит **массы** `m_i` и **удельные внутренние энергии** `u_i` по узлам.

Физически сохраняемая величина для узла — это **внутренняя энергия**:

- `U_i = m_i * u_i`

В исходной реализации `m` и `u` интегрировались раздельно явным RK2 (Heun):

- `m_{n+1} = m_n + 0.5*dt*(dm0 + dm1)`
- `u_{n+1} = u_n + 0.5*dt*(du0 + du1)`

Даже если в непрерывной постановке выполнено `dU/dt = m*du/dt + u*dm/dt`, дискретно при раздельной интеграции появляется перекрёстный член порядка `O(dt^2)`:

- `U_{n+1} = (m_{n+1}*u_{n+1}) != U_n + ∫ dU/dt dt`

В результате **баланс энергии** (который в этой модели явно проверяется) показывал заметную ошибку.

### 2) Отрицательная «энтропия смешения»
В разложении генерации энтропии использовался термин вида `ṁ * (s_node(dn) - s_throttle)`.

Такой член может быть отрицательным (например, когда поток из более тёплого узла попадает в более холодный). Он скорее похож на **изменение энтропии входящего потока**, но не гарантирует неотрицательность как генерация.

### 3) Отрицательная «энтропия по элементам»
Для дросселей/сопротивлений использовалась оценка `ṁ*(s_dn - s_up)` на **узловых** энтропиях.

Это смешивает в одном термине:

- дросселирование (падение давления) и
- последующее «дотягивание» температуры/состояния до состояния downstream‑узла.

В итоге знак мог становиться отрицательным, что физически некорректно именно для **генерации энтропии в сопротивлении**.

### 4) Дубликаты в `return` при `record_full=True`
Во всех `model_pneumo_v*.py` возвращались дубликаты датафреймов (`df_energy_edges`, `df_energy_groups`, `df_atm`) — фактически лишние элементы в кортеже.

Это не всегда ломало код (часто делали `out[:11]`), но создаёт риск скрытых ошибок и путаницу.

## Что сделано

### A) Энерго‑консервативный RK2‑апдейт `u` через `U=m*u`
Фикс реализован в `model_pneumo_v8_energy_audit_vacuum.py`:

1. В `rhs()` добавлен режим `return_aux=True`, который возвращает `dU_dt` по узлам (то есть физический `dU/dt` из потоков + `-p*dV` + теплообмен).
2. В RK2‑шаге интегрируются:
   - `m` как обычно (Heun)
   - `U` как `U_{n+1}=U_n+0.5*dt*(dU0+dU1)`
3. После этого `u_{n+1}` **переопределяется** как `u = U / max(m, m_eps)`.

Параметр:

- `энерго_консервативный_апдейт_u` (bool, по умолчанию `True`).

Добавлены диагностические поля в `df_atm`:

- `численная_коррекция_u_проекция_Дж`
- `численная_коррекция_u_клип_Дж`

### B) Энтропия смешения теперь неотрицательная
Термин «mixing» заменён на гарантированно неотрицательную оценку через «thermalization to reservoir»:

- `Sgen = ṁ * cp(T_dn) * (r - 1 - ln r)`, где `r = T_up / T_dn`.

Этот выражение ≥ 0 для `r>0`.

### C) «Энтропия по элементам» считается через дросселирование
Теперь для сопротивления берётся выходная энтропия **после дросселирования**, но при температуре upstream:

- `s_out = s(p_dn, T_up)`

То есть учитывается именно рост энтропии за счёт падения давления (в рамках принятой в модели идеализации).

### D) Единый формат `return` при `record_full=True`
Во всех `model_pneumo_v*.py` исправлено:

- теперь возвращается ровно 11 элементов:
  1. `df_main`
  2. `df_drossel`
  3. `df_energy`
  4. `nodes`
  5. `edges`
  6. `df_energy_edges`
  7. `df_energy_groups`
  8. `df_atm`
  9. `df_p`
  10. `df_mdot`
  11. `df_open`

### E) Расширен `self_check.py`
Добавлены проверки:

- относительная ошибка баланса энергии (`ошибка_энергии_газа_отн`)
- неотрицательность `энтропия_смешение_Дж_К`
- формат выхода `record_full` (len == 11)
