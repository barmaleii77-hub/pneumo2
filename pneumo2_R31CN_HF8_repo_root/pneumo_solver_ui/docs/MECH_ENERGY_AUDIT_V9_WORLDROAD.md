# Механо‑энергоаудит v9 (double wishbone + worldroad)

## Зачем
Энергоаудит — это быстрый способ ловить математические ошибки, которые неочевидны по графикам перемещений:
- перепутанный знак силы/скорости;
- несогласованность `p·dV` с силой цилиндра;
- неправильный gating контакта (пружина «работает» без контакта), что порождает систематический дрейф энергии;
- ошибки в мэппинге «шток → вертикальная сила» через виртуальную работу.

## Баланс
В релизе реализован P0‑баланс механики:

- Механическая энергия:
  - кинетическая: кузов + 4 колеса + вращение (крен/тангаж)
  - потенциальная: гравитация + табличная пружина + шина + отбойники

- Источники/стоки мощности:
  - `P_road = Σ F_tire * z_road_dot` (ввод энергии дорогой, base‑excitation)
  - `P_axay = M_roll * phi_dot + M_pitch * theta_dot` (вклад внешних `ax/ay`)
  - `P_gas = Σ (p_gauge * dV/dt)` (работа газа на механику через цилиндры)
  - `D_tire >= 0`, `D_stop >= 0` (диссипация)

Целевая проверка:

`E_mech(t) - E_mech(0)  ≈  ∫ (P_road + P_axay + P_gas - D_tire - D_stop) dt`

Ошибки логируются в:
- `ошибка_энергии_мех_Дж`
- `ошибка_энергии_мех_отн`

## Независимая проверка p·dV
В модели также считается мощность через механику штока:
- `P_force = Σ F_cyl_gauge * s_dot`

И сравнивается с газовой формой:
- `P_pdv = Σ p_gauge * dV/dt`

Ошибка:
- `ошибка_мощности_p_dV_Вт = P_pdv - P_force`

Если знаки и мэппинг верны, ошибка должна быть близка к нулю (с погрешностью интегратора).

## Что считается «нормой»
- smoke‑уровень: `max|ΔE|/E0 < 0.25`, `max|p·dV - ΣF·ṡ| < 0.5 Вт` (только чтобы ловить грубые баги)
- для регрессии (P1) пороги обычно ужесточаются на выбранном наборе тестов.

## Примечание по gauge/absolute
Силы цилиндров в механике строятся по избыточному давлению `(p_abs - Patm)`. Поэтому и энергообмен «газ → механика» в P0‑аудите ведётся в gauge‑форме. Для контроля разницы gauge/absolute в отчётах остаётся диагностический канал (если включён в конкретной модели).
