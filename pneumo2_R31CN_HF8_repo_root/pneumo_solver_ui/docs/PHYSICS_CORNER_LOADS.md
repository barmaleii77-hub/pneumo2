# PHYSICS: статическое распределение веса по углам (CG → corner loads)

## Зачем это нужно
В «нулевой позе» (t=0, ровная дорога z=0) модель должна стартовать из физически
правдоподобного статического состояния:
- колесо находится на дороге (контакт без «подскока»);
- шина имеет реалистичную статическую деформацию под нагрузкой;
- пружины/пневмоцилиндры предварительно нагружены так, чтобы рама не имела
  огромного начального ускорения.

Ранее (в базовой версии) нагрузка рамы распределялась равномерно как `m_body*g/4`.
Для более реалистичной модели введено распределение веса по углам в зависимости
от положения центра тяжести (CG).

## Принятая система координат
- Ось **x** направлена вперёд.
- Ось **y** направлена влево.
- Нулевая точка (0,0) в плане: середина между осями и продольная ось автомобиля.
- Порядок углов (corner_order): **[ЛП, ПП, ЛЗ, ПЗ]**.

## Параметры
- `база` / `база_м` — wheelbase, L
- `колея` / `колея_м` — track, T
- `cg_x_м` — смещение CG вдоль базы относительно середины:
  - `cg_x_м > 0` → CG смещён вперёд
- `cg_y_м` — смещение CG поперёк относительно продольной оси:
  - `cg_y_м > 0` → CG смещён влево

Совместимость:
- `x_cg_от_передней_оси_м` / `x_cg_from_front_m` (используется в некоторых
  старых подсистемах). Если `cg_x_м` не задан, то делаем перевод:
  `cg_x = wheelbase/2 - x_from_front`.

## Формула (упрощённая)
Полная задача реакции четырёх опор без учёта жёсткости — статически
неопределима. Для инициализации и целевых усилий принят простой separable-ансамбль:

1) Доля нагрузки на переднюю/заднюю ось:
- `front_frac = 0.5 + cg_x/L`
- `rear_frac  = 0.5 - cg_x/L`

2) Доля нагрузки на левую/правую сторону:
- `left_frac  = 0.5 + cg_y/T`
- `right_frac = 0.5 - cg_y/T`

3) Нагрузки по углам (N):
- `F_ЛП = W * front_frac * left_frac`
- `F_ПП = W * front_frac * right_frac`
- `F_ЛЗ = W * rear_frac  * left_frac`
- `F_ПЗ = W * rear_frac  * right_frac`

где `W = m_body*g`.

Безопасность:
- доли `cg_x/L` и `cg_y/T` клиппируются по модулю до 0.49 (чтобы не получить
  отрицательные реакции при неадекватных вводах). В отчёт попадает `cg_msg`.

## Где используется в коде
Модуль: `pneumo_solver_ui/corner_loads.py`
- `compute_body_corner_loads(...)` → `F_body_corner` + отчёт
- `parse_cg_offsets_with_geom(...)` → извлечение параметров CG из `params`

Обновлены модели:

- `model_pneumo_v9_doublewishbone_camozzi.py`

## Режимы распределения (v659)

Начиная с **v659** добавлен параметр `corner_loads_mode` (алиасы: `режим_нагрузки_углы`):

- `corner_loads_mode = "cg"` *(по умолчанию)* — старый separable‑режим (front/rear × left/right), как описано выше.
- `corner_loads_mode = "stiffness"` — «разрешение» статической неопределимости через **эффективные вертикальные жёсткости углов** `k_i` (N/m) и решение задачи **минимума энергии деформации**.

### Идея stiffness‑режима
Считаем, что рама опирается на 4 «пружины к земле» с жёсткостями `k_i` и координатами углов `(x_i, y_i)`:

- линейная модель малых углов:
  - `δ_i = z + φ*y_i - θ*x_i`
  - `F_i = k_i * δ_i`

где:
- `z` — heave (м),
- `φ` — roll (рад),
- `θ` — pitch (рад),
- `F_i` — вертикальная реакция угла на раму (Н).

Условия равновесия (в обобщённых координатах) записываются как:

- `A^T * F = [W, W*cg_y, -W*cg_x]`,
- `A = [1, y, -x]` (матрица 4×3),
- `K = diag(k_i)`.

Отсюда получаем линейную систему 3×3:

- `(A^T K A) * q = Q`,
- `q = [z, φ, θ]^T`.

Таким образом, при заданных `k_i` получаем **единственный** набор `F_i`, который:
- удовлетворяет сумме сил и моментам,
- выбирается так, чтобы энергия `0.5 * Σ k_i δ_i^2` была минимальна.

### Параметры жёсткостей
Жёсткости можно задать:

1) Вектором (предпочтительно):
- `k_corner_N_m = [k_FL, k_FR, k_RL, k_RR]`

2) По углам (пример):
- `k_corner_ЛП_Н_м`, `k_corner_ПП_Н_м`, `k_corner_ЛЗ_Н_м`, `k_corner_ПЗ_Н_м`


3) **Авто‑оценкой** (опционально, если пользователь не задал `k_corner_*`):

- `corner_stiffness_auto = true|false` *(по умолчанию: false)* — оценить эффективные жёсткости углов из
  табличной характеристики пружины и кинематики (motion ratio), а затем объединить с жёсткостью шины как
  последовательные элементы:

  - `k_susp_i ≈ k_spring_rod * (drod/ddw|_0)^2` *(или аналогичный коэффициент механики)*
  - `k_corner_i = (k_tire * k_susp_i) / (k_tire + k_susp_i)`

- `corner_stiffness_ref_x_m` — опорная компрессия (м) для вычисления наклона `dF/dx` табличной пружины.
  По умолчанию берётся `0.5 * max(tab_x)`.

Ограничения/допущения авто‑оценки:

- **Пружина** берётся как локальная линеаризация табличной характеристики в точке `x_ref`.
- **Пневматика** может быть (опционально) учтена как малосигнальная газовая жёсткость цилиндров ("sealed"‑допущение: масса в полостях постоянна, `p·V^n≈const`).

  Параметры:
  - `corner_stiffness_include_pneumo = true|false` *(по умолчанию: true, но работает только если `corner_stiffness_auto=true`)*
  - `corner_pneumo_stiffness_n` *(по умолчанию: 1.4)* — показатель политропы
  - `corner_pneumo_stiffness_volume_factor` *(по умолчанию: 1.0)* — множитель эффективного объёма (">1" уменьшает жёсткость)
  - `corner_pneumo_stiffness_scale` *(по умолчанию: 1.0)* — общий коэффициент
  - `corner_pneumo_stiffness_ref_s_frac` *(по умолчанию: 0.5)* — относительное положение штока для оценки объёмов

  Формула (double‑acting цилиндр):
  - `k_gas_ax ≈ n * P_ref * (A_cap^2/V_cap + A_rod^2/V_rod)`
  - `k_gas_wheel ≈ k_gas_ax * (ds/ddw|_0)^2` *(или `(drod/ddw|_0)^2` в MR‑моделях)*

- Не учитываются bump‑stops и упругость/регулирование клапанов (regulator compliance): если реальный контур поддерживает давление почти постоянным, фактическая жёсткость будет ниже, чем при sealed‑допущении.
- Авто‑оценка предназначена для инициализации нулевой позы (дальше динамика + `static_trim` уточняют равновесие).

Если `corner_loads_mode="stiffness"`, но пользователь не задал `k_corner_*`, по умолчанию используется `k_tire`
или (если `corner_stiffness_auto=true`) — авто‑оценка. Если жёсткости не удалось получить корректно,
код сделает **безопасный fallback** к `cg` (см. `cg_msg` / `cg_json`).

### Диагностика
В `df_atm` добавлены диагностические поля (через `cg_json`):
- `mode` — фактически применённый режим,
- `cross_weight_frac` — `(F_ЛП + F_ПЗ) / W`,
- `diag_bias_N` — `(F_ЛП + F_ПЗ) - (F_ПП + F_ЛЗ)`,
- `k_corner_N_m`, `q_*`, `cond_M` (для stiffness‑режима).

> Важно: это используется **только для инициализации нулевой позы** и целевых преднатягов.
> Точная реакция затем определяется динамической моделью + `static_trim` (если включён).
