# v6.42 — p0_override и режимы подбора предсжатия пружины x0

## Зачем это нужно

В нулевой позе проекта (t=0):
- дорога плоская, уровень 0,
- подвеска в статическом положении,
- штоки цилиндров в середине хода (`s0 = stroke/2`),
- нулевая база для графиков/анимации.

В v9 worldroad статическая равновесность обеспечивается подбором предсжатия пружины `x0` так, чтобы:
- по каждой вертикальной цепочке «рама → (пружина + цилиндры) → колесо → шина → дорога» суммарные силы давали нулевые ускорения,
- колесо в t=0 не имело ускорения `zw_ddot` на плоской дороге.

Проблема:
- если начальные давления в пневмосистеме **асимметричны** по углам (например, один цилиндр имеет другое давление), то «один x0 на все углы» (
  базовое поведение) даёт корректный баланс кузова по сумме сил, **но** отдельные колёса могут стартовать с ненулевым `zw_ddot`.

Решение v6.42:
1) `p0_override` — точечно задавать начальные абсолютные давления `p0` по узлам;
2) `пружина_x0_режим` — выбирать, как подбирать `x0`:
   - `global` (по умолчанию, как в базе)
   - `per_axle`
   - `per_corner` (наиболее физически «жёсткий» для нулевой позы)
   - `manual` (если `пружина_x0_м` задан явно)

---

## 1) p0_override: задание начальных давлений по узлам

Где задавать:
- `test['p0_override']` (удобно для тестов/оптимизации)
- или `params['p0_override']`

Формат:
```python
p0_override = {
  'ИмяУзла': p_abs_Па,
  'Шаблон*': p_abs_Па,
}
```

Поддерживаются:
- точные имена узлов,
- `*`-шаблоны (через `fnmatch`), например `"Ц1_*_ШП"`.

Пример (поднять давление в штоковой полости Ц1 на левом переднем углу):
```python
test = {
  'p0_override': {
    'Ц1_ЛП_ШП': 101325.0 + 8e5,
  }
}
```

В `df_atm` логируется:
- `p0_override_applied` — сколько присваиваний было применено.

## 2) Режимы подбора x0 (предсжатие пружины)

Параметр:
- `пружина_x0_режим` (алиас: `spring_x0_mode`).

Возможные значения:
- `global` — один `x0` на все 4 угла (поведение базы)
- `per_axle` — отдельные `x0` для передней и задней оси
- `per_corner` — отдельный `x0` на каждый угол

Ручной режим:
- если задан `пружина_x0_м` (алиас `spring_x0_m`), режим автоматически становится `manual`.
  Можно задать число (одно значение на все углы) или список длиной 4.

Что пишется в `df_atm`:
- `spring_x0_mode` — фактически применённый режим
- `spring_x0_override_used` — 1 если использован ручной `пружина_x0_м`
- `spring_x0_mean_m/min_m/max_m`
- `spring_x0_ЛП_м`, `spring_x0_ПП_м`, `spring_x0_ЛЗ_м`, `spring_x0_ПЗ_м`

### Рекомендация для нулевой позы
- Если вы хотите **жёстко** выполнять требование «штоки в середине хода и статика без ускорений» даже при асимметрии давлений — используйте:

```json
{
  "пружина_x0_режим": "per_corner"
}
```
