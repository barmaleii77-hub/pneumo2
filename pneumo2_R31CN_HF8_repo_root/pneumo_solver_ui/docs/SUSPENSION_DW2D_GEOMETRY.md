# Геометрия подвески DW2D (двойные поперечные рычаги, упрощённая 2D-модель)

Этот документ описывает, **как в приложении задаётся и проверяется геометрия подвески** для режима
`механика_кинематика = dw2d / dw2d_mounts`.

> Важное: текущая реализация **не является полной 3D-кинематикой double-wishbone**.
> Это *упрощение в плоскости (Y–Z)*, которое даёт **аналитическую/дифференцируемую связь**
> `dw → Δrod → d(Δrod)/d(dw)` и хорошо подходит для «пневматика + вертикальная динамика».

---

## 1. Где задаётся геометрия в интерфейсе

Откройте страницу:

**Меню слева → Проверки и настройка → Геометрия подвески (DW2D)**

Там вы:
- вводите параметры геометрии;
- видите графики `Δrod(dw)` и `MR(dw)`;
- получаете предупреждения/ошибки валидации;
- сохраняете параметры в рабочую конфигурацию.

Дополнительно: на основной странице «Интерфейс» рядом с параметром `механика_кинематика`
есть подсказка и кнопка перехода на страницу геометрии.

---

## 2. Что такое `dw` и `Δrod`

- `dw` — ход колеса относительно угла/рамы (в вертикали), **сжатие = dw > 0**.
- `Δrod(dw)` — ход штока (или изменение эффективной длины цилиндра), **сжатие = Δrod > 0**.
- В модели цилиндр хранит переменную хода `s` (0…Stroke), и используется связь:

` s = s0 - Δrod `

где `s0` — положение штока на нулевой позе (`dw=0`). По умолчанию
`s0 = 0.5 * Stroke`, то есть **нуль-поза соответствует середине хода штока**.

---

## 3. Параметры DW2D (смысл)

Параметры задаются **в метрах** и описывают геометрию в плоскости Y–Z:

- `track_m` — колея (расстояние между центрами колёс L/R).
- `pivot_inboard_m` — насколько внутренний шарнир нижнего рычага смещён от центра колеса
  внутрь по Y.
- `pivot_z_m` — высота внутреннего шарнира нижнего рычага (в координате Z модели).
- `lower_arm_len_m` — длина нижнего рычага (в плоскости Y–Z).
- `low_frac` — доля по нижнему рычагу от внутреннего шарнира до точки крепления штока (0…1).
- `top_sep_m` — расстояние между верхними точками крепления штока слева/справа (по Y).
- `top_z_m` — высота верхней точки крепления штока (по Z).

Для левой стороны используется `sign_lr = -1`, для правой `+1`.

---

## 4. Математика (кратко)

Схема:
1) Нижний рычаг задаётся как жёсткий отрезок длиной `L` между внутренним шарниром и шаровой опорой.
2) Шаровая опора (upright joint) перемещается по Z на `dw`.
3) Из условия постоянства длины рычага вычисляется новая координата по Y (решение окружности).
4) Точка крепления цилиндра на рычаге берётся на доле `low_frac`.
5) Длина цилиндра `Lc(dw)` считается как расстояние между нижней и верхней точками крепления.
6) `Δrod = sign * (Lc0 - Lc(dw))` выбирается так, чтобы при `dw>0` получалось `Δrod>0`.
7) Производная `d(Δrod)/d(dw)` считается аналитически.

Ограничение реализуемости:
- выражение `L^2 - dz^2` должно быть ≥ 0, иначе геометрия невозможна для данного `dw`.

---

## 5. Встроенные проверки (валидация)

### 5.1. Статическая валидация параметров
Страница «Геометрия подвески (DW2D)» проверяет:
- корректность диапазонов (например `0 < low_frac < 1`);
- реализуемость геометрии на заданном диапазоне `dw`;
- отсутствие NaN/Inf;
- адекватность `MR(dw)` (производная не должна «взрываться»).

### 5.2. Проверка по фактическому диапазону симуляции
После прогона теста выполняется autoself-проверка `dw2d_dynamic_range`:
- берётся реальный диапазон `dw` из результатов;
- на нём проверяется реализуемость геометрии и отсутствие деградации.

Это важно: иногда геометрия корректна на «малых ходах», но ломается на больших.

---

## 6. Что не моделируется (честно)

DW2D не учитывает:
- верхний рычаг как отдельный кинематический контур;
- развал/кастер/скраб-радиус/траекторию центра колеса в 3D;
- кинематику рулевого управления;
- прогиб/эластокинематику сайлентблоков.

Если понадобится — следующий шаг: расширить кинематику до 3D (включая оба рычага)
и получать `dw`/`camber`/`toe`/`kingpin` из системы уравнений связей.
