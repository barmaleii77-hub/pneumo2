# (DEPRECATED) Interfeis764Release58 — UI refactor applied to RealizatsiyaOptimizatsiiRelease58

Эта папка = исходный релиз **RealizatsiyaOptimizatsiiRelease58**, но с применённым «человеко‑ориентированным» UI‑слоем (прогрессивное раскрытие, master→detail вместо широких таблиц).

Цель: сделать UI **удобным для человека**, без:
- горизонтального скролла,
- таблиц на 100 параметров на одном экране,
- дублирующих мест ввода,
- «миллиона чекбоксов/вкладок».

## 2) Что именно изменено (суть UI-рефакторинга)

### 2.1 Прогрессивная навигация (вместо «простыни»)
В верхней части экрана добавлено:
- **Режим UI**: `Рабочее место` / `Полный интерфейс`
- В «Полном интерфейсе»: 6 разделов
  1) Модель  
  2) Параметры  
  3) Тесты  
  4) Прогон  
  5) Результаты  
  6) Инструменты  

Идея: **один экран = одна задача**.  

### 2.2 Перенос настроек из сайдбара
Сайдбар «разгружен». В нём остаются только небольшие UX‑тумблеры.
Выбор файлов проекта и параметры оптимизации перенесены в соответствующие разделы.

### 2.3 Параметры: вместо широкой таблицы → «список + карточка параметра»
- Поиск/фильтр по группе
- Опция «только оптимизируемые»
- Лимит количества карточек на экране (производительность)
- Карточка параметра:
  - значение: number_input / slider (если применимо)
  - `Оптимизировать`: checkbox
  - диапазон min/max скрыт в expander «Диапазон оптимизации»

### 2.4 Тест‑набор (suite): вместо широкой таблицы → «список тестов + карточка теста»
- Слева: список с поиском
- Кнопки: добавить / дублировать / удалить
- Справа: карточка выбранного теста с релевантными полями и слайдерами
- Импорт/экспорт suite.json в expander

### 2.5 Правило «нет горизонтального скролла» для таблиц результатов
Функция `safe_dataframe()` переработана:
- если таблица **слишком широкая**, показываем только первые N колонок (preview)
- полная строка доступна как «карточка строки» в expander

## 5) UX-основания (коротко)

В этом релизе UI приведён к типовым паттернам для «сложных инженерных интерфейсов»:
- progressive disclosure (прогрессивное раскрытие)
- master→detail для таблиц (список/поиск + карточка строки)
- минимизация горизонтального скролла

Ссылки (для ориентира):
- NN/g: Progressive Disclosure — https://www.nngroup.com/articles/progressive-disclosure/
- NN/g: Data Tables: Four Major User Tasks — https://www.nngroup.com/articles/data-tables/
- NN/g: Beware Horizontal Scrolling — https://www.nngroup.com/articles/horizontal-scrolling/
