Что нового в v3 (сверху v2)
1) «Контакт‑потерян» и «минимальный зазор до отрыва» теперь в KPI по КАЖДОМУ колесу:
   - контакт_потерян_<угол> (0/1) и доля_времени_отрыв_<угол>
   - мин_зазор_до_отрыва_<угол>_Н и мин_зазор_до_отрыва_<угол>_%стат
   Эти метрики есть:
   - по каждому тесту (с префиксом <тест>__...)
   - и в сводном виде по всем тестам (свод__...)

2) «Запас до пробоя по крену/тангажу» введён как НЕПРЕРЫВНАЯ метрика, а не стоп‑условие:
   - критический_крен_град / критический_тангаж_град (геометрия)
   - запас_до_пробоя_крен_град / запас_до_пробоя_тангаж_град (+ варианты в %)
   Ролловер как аварийная остановка остаётся, но только по «жёсткому» пределу
   (аварийный_предел_крен_град/аварийный_предел_тангаж_град), чтобы не портить геометрию.

3) Геометрия вертикальных перемещений углов рамы сделана более «не‑малой‑угловой»
   (sin/cos вместо линейного phi*y - theta*x), чтобы модель не ломалась при больших кренах.

Остальное (из v2)
1) KPI держит «запасы» (не булевы флаги) по:
   - ходу штока: мин_запас_до_упора_штока_<угол>_м и ..._все_м
   - скорости штока: макс_скорость_штока_<угол>_м_с и ..._все_м_с
2) Энергетический аудит:
   - доля_энергии_выхлоп / дроссели / клапаны / регуляторы
   + доля_энергии_макс_категория (быстрый индикатор «вся энергия в одном месте»)
3) «Раньше‑жёстко»: метрика времени успокоения крена (settle time).
4) Pareto‑выбор: TOP‑10 + 3 финала (aggressive/balanced/comfort).

Единицы в UI (важно)
- Давления в UI задаются в "атм (изб.)" (gauge). Внутри модели всё хранится в Па (абсолютное):
    P_abs = P_atm + P_gauge_atm * 101325
- Объёмы ресиверов/аккумулятора в UI задаются в литрах (л), объём линий — в миллилитрах (мл).
  Внутри модели объёмы хранятся в м³.
