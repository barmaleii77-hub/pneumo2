# PneumoApp_v6_80_R176_WINDOWS_CLEAN_R25_2026-03-20

## Что вошло
- Desktop Animator:
  - 3D перенесён в центральную область окна; центральная белая пустая панель убрана.
  - Кнопки управления панелями перенесены на верхнюю toolbar.
  - После первого показа окна панели раскладываются по экрану по `QScreen.availableGeometry()`/DPI-метрикам.
  - Добавлен общий сброс масштаба/вида для 2D/3D.
- Road / contact:
  - поперечная форма дороги перестроена с квадратичной на piecewise-linear по right/center/left профилям;
  - пятно контакта теперь отбирается как уточнённый subset road mesh внутри объёма колеса, а не как oversized whole-face grab;
  - контактная связь колесо→дорога строится по локальному `wheel_up`, а не по фиксированному `center.z - radius`.
- Cylinders / pistons:
  - тела цилиндров сделаны прозрачнее;
  - поршни и маркеры поршней сделаны заметнее.
- Web UI / launcher:
  - ключевые редактируемые expanders открываются по умолчанию, чтобы не схлопываться на каждом rerun;
  - запуск Desktop Animator из web/launcher предпочитает `pythonw.exe` на Windows, чтобы не плодить лишнее консольное окно.

## Честные ограничения
- Это всё ещё reduced-DW runtime, а не полная 3D hardpoint-физика подвески.
- Форма road/contact стала ближе к физике и геометрии, но остаётся визуализацией по текущему road mesh/export contract.
- Геометрия цилиндров/поршней остаётся service-derived из доступных хода/диаметров/мёртвых объёмов; отдельной полной CAD-геометрии крышек/штока/головок в модели всё ещё нет.
